## Codebase Patterns
- `services/recorder` uses a **src layout** (`services/recorder/src/recorder/…`); future files (schema.py, writer.py, session.py, etc.) go under `src/recorder/`.
- The recorder package is installed as a uv workspace member (`trace-recorder`) and should be imported as `from recorder.xyz import …`, NOT `services.recorder.recorder.xyz`.
- Root `pyproject.toml` owns shared tooling config (ruff/mypy/pytest); per-service `pyproject.toml` only declares that service's runtime deps + hatchling build.
- Ruff's isort needs `known-first-party = ["recorder", "synthesizer", "runner", "gateway"]` or it misgroups the workspace imports in gateway/main.py.
- PyObjC wheels only exist for macOS — deps are marked `; sys_platform == 'darwin'` so `uv sync` still works if Ralph ever runs on non-darwin.
- `mypy --strict` has PyObjC module overrides in root pyproject (`Cocoa.*`, `Quartz.*`, `ApplicationServices.*`, `AppKit.*`, `Foundation.*`, `objc.*`); add new PyObjC module imports here, not in each file.
- Quality-gate commands for the recorder module are:
  - `uv run ruff check services/recorder`
  - `uv run mypy --strict services/recorder`
  - `uv run pytest tests/recorder/`
- Pre-existing gateway/integration test failures (`ModuleNotFoundError: gateway`) are NOT caused by R-001 and are outside this branch's scope — a future `feat/integration` story will install gateway as a proper package.
- **Schema validation**: `recorder.schema` exposes `validate_event(event)` / `validate_metadata(meta)` backed by `jsonschema`'s Draft 2020-12 validator (with `FORMAT_CHECKER` enabled for `uuid`/`date-time`). Future writer/session code should call these before writing JSON to disk so bad events never land in `events.jsonl`.
- **Schema file lookup**: `recorder.schema.find_schema_path()` walks up from `__file__` to find `contracts/trajectory.schema.json`. Works for the uv-workspace editable install. If the contracts ever move or get bundled into the wheel, update this function — do NOT hardcode a path.
- **Local `$ref` only**: the `load_event_schema()` / `load_metadata_schema()` wrappers build standalone schemas with a `#/$defs/...` fragment ref (no `$id`), which keeps `jsonschema` from trying to HTTP-fetch the schema's `$id` URL at validate time. If you add new top-level sub-schemas for Synthesizer/Runner to reuse, follow the same pattern.
- **TrajectoryWriter ownership**: `recorder.writer.TrajectoryWriter` is the only module allowed to mutate the on-disk trajectory directory. All writes go through its lock; session/orchestrator code must not write `events.jsonl` / `metadata.json` / `screenshots/*` directly. It validates via `recorder.schema.validate_{event,metadata}` BEFORE any disk write, so bad data never leaks to disk. The constructor creates `<root>/<uuid>/` + `screenshots/` with perms `0700` and the writer is usable as a context manager (`__exit__` calls `close()`).
- **Atomic JSON writes**: `_atomic_write_json(path, data)` writes to `<path>.tmp`, fsyncs, and renames. Use it for any metadata-style file. `events.jsonl` is append-flush (no fsync per line — documented v1 tradeoff).
- **Datetime import style**: use `from datetime import UTC, datetime` (Python 3.11+). Ruff's `UP017` flags `timezone.utc` — don't revive it. ISO strings are produced with `datetime.now(UTC).isoformat(timespec="milliseconds")` which yields `+00:00` suffixes; the schema's `format: date-time` accepts both `+00:00` and `Z`.
- **Ruff SIM105**: prefer `with contextlib.suppress(OSError): os.fsync(...)` over `try/except/pass` for best-effort fsync on filesystems that don't support it (tmpfs in pytest tmp_path).
- **PyObjC mocking in tests**: for any module that does `from ApplicationServices import X` / `from Quartz import Y` inside a function, tests stub the framework by installing a `types.ModuleType("Quartz")` / `types.ModuleType("ApplicationServices")` into `sys.modules` via `monkeypatch.setitem`. The import inside the tested function then resolves to the stub. For ImportError coverage, install a stub that is missing the expected attributes — `from X import missing_attr` raises ImportError, which the module under test should catch.
- **Permission-check pattern**: `recorder.permissions` keeps each macOS framework call in its own thin `_check_*` helper, and `check_permissions()` / `get_missing_permissions_error()` simply compose them. Tests patch the helpers for combination coverage and patch the PyObjC symbols inside `sys.modules` for helper-level coverage. Use the same pattern for any future PyObjC-backed module (event_tap, ax_resolver, focus_tracker, screenshot) to keep the suite hermetic on Ralph's sandbox.
- **`@pytest.mark.macos` is opt-in only**: `tests/recorder/conftest.py` auto-skips every test tagged `macos` unless `TRACE_RUN_MACOS_TESTS=1` is set. Use this marker for ANY test that needs real PyObjC framework behaviour (CGEventTap, AXUIElement, CGWindowListCreateImage, etc.). Hermetic unit coverage should still be provided alongside via `sys.modules`-stubbed Quartz/ApplicationServices so the default run stays green.
- **Splitting PyObjC callback from normaliser**: for modules driven by a C-callback (CGEventTap), put the pure event→dict conversion in a module-level function that takes the framework module as a parameter (`_normalise_event(code, event, quartz)`), and expose a thin `handle_cg_event(code, event)` method on the class that the tap callback delegates to. Tests exercise the normaliser + re-enable logic by calling `handle_cg_event` with fake events — no run-loop or thread needed. `event_tap.py` is the reference implementation.
- **Re-enabling a disabled tap**: `CGEventTapEnable(tap, True)` re-arms a tap the OS disabled via `kCGEventTapDisabledByTimeout` / `kCGEventTapDisabledByUserInput`. Both constants appear as unsigned `uint32` (0xFFFFFFFE / 0xFFFFFFFF) in headers but PyObjC sometimes surfaces them as signed ints (-2 / -1); match both forms.
- **Bounded-time framework calls**: when a PyObjC call could block indefinitely (AX queries, inter-app RPC), wrap it with a daemon `threading.Thread` and a `threading.Event` set in a `finally`. The caller `event.wait(timeout)` — on timeout the worker thread just keeps living as a daemon. Do not rely on `Thread.join(timeout)` alone to tell "done" from "still running"; always use a separate event sentinel. See `ax_resolver.resolve_element_at` for the reference.
- **PyObjC AX value shapes**: AX position/size attributes come back as one of three shapes depending on macOS/PyObjC version — native 2-tuple, bridged object with `.x/.y` or `.width/.height`, or raw `AXValueRef` that must be unwrapped via `AXValueGetValue(value, kAXValueCG{Point,Size}Type, None) -> (success, unpacked)`. Any new AX-consuming code should tolerate all three (see `_unpack_ax_value` in ax_resolver.py).
- **NSWorkspace subscription pattern**: prefer `addObserverForName:object:queue:usingBlock:` (a Python callable passed as block) over the NSObject-subclass selector pattern — it keeps the handler method on the Python class and avoids `@objc.selector` machinery. The block returns an opaque observer handle that must be passed back to `removeObserver:` on stop. Tests stub `AppKit.NSWorkspace.sharedWorkspace().notificationCenter()` with a class that captures `(name, block)` in a list so we can invoke the block synchronously with a fake notification. See `focus_tracker._subscribe_workspace` for the reference.
- **NSDictionary dual-access**: `userInfo()` on an `NSNotification` is bridged to a Python dict in most PyObjC builds but not all. Accept both by trying `.get(key)` first (callable on a bridged dict) and falling back to `objectForKey_(key)` on an ObjC-only stub. Same pattern applies to any NSDictionary the recorder reads out of a framework callback.
- **Idle-flush timer pattern**: for any module that aggregates into a buffer until a "nothing happened for N seconds" signal, use a `threading.RLock` around the state + a `threading.Timer` that is cancelled-and-replaced on every ingest event. The RLock (not a plain Lock) is required because the Timer's callback may take the lock while a caller-driven `flush()` is mid-flight. The Timer callback should also check an `is_stopped` flag so a late-firing cancelled timer is a silent no-op. `text_aggregator.py` is the reference.
- **Session-layer translation of event_tap output**: `recorder.event_tap` emits modifier names in PyObjC framework style (`"command"`, `"control"`, `"option"`, …). The trajectory schema's `modifier` enum uses short forms (`"cmd"`, `"ctrl"`, `"opt"`, …). Translation happens in the session orchestrator (R-010) when shaping a schema-valid `keypress` event. Downstream consumers of event_tap (text_aggregator, session) must agree on which name style they expect — currently everything inside the recorder uses the long PyObjC names; ONLY the session's final schema-event builder maps to the short enum.
- **Stateless policy modules**: `recorder.keyframe_policy.KeyframePolicy` is stateless — the session passes `(event_type, seconds_since_last_keyframe)` each call and the policy answers yes/no + reason. The session owns the "time since last keyframe" state and the periodic timer. Prefer this shape for any future rule-based gate (e.g. scroll-debounce) — stateless is easier to test, thread-safe by construction, and doesn't couple the rule to a particular timer impl.
- **Class-level state in PyObjC test doubles**: when emulating the `cls.alloc().init…_(…)` two-step idiom, do NOT use mutable class attributes at the base class (RUF012). Either put state on the instance, or declare `ClassVar[...]` explicitly, or create a fresh subclass per test via `type("Name", (Base,), {...})` — the screenshot stub uses the subclass trick so each test's `rep_calls` list is isolated.
- **PNG validation without Pillow**: the `recorder.screenshot` module exposes `PNG_MAGIC = b"\\x89PNG\\r\\n\\x1a\\n"` and rejects bytes that don't start with it. Tests synthesize a minimal valid PNG by hand (magic + IHDR + one compressed IDAT + IEND, with `zlib.crc32` for the per-chunk CRC) — avoid adding Pillow just to verify PNG shape. Dimension parsing uses `struct.unpack(">II", png[16:24])` once you've asserted `png[12:16] == b"IHDR"`.
- **RecordingSession owns `seq`**: `recorder.session.RecordingSession` is the single place where `seq` values are minted for a running trajectory. `TrajectoryWriter` still has its own auto-seq counter (belt-and-suspenders), but every normal emit path passes a pre-assigned seq under the session's `_lock`. Any future capture path that does NOT go through the session (e.g. an HTTP admin endpoint that injects events) must either acquire the same lock or accept interleaved seqs.
- **Session dependency injection**: every macOS-dependent piece of `RecordingSession` is injected via a constructor kwarg (`event_tap_factory`, `focus_tracker_factory`, `text_aggregator_factory`, `permissions_check`, `ax_resolver`, `screenshot_capturer`, `display_info_provider`, `keyframe_policy`). Hermetic tests pass fakes; real use passes nothing and defaults fall back to the real modules. Keep this pattern for any future orchestrator — it is the only way tests can run in Ralph's sandbox without PyObjC.
- **Dual-lock orchestrator pattern**: `_start_stop_lock` serialises `start()`/`stop()` bookkeeping; `_lock` guards per-event state (seq, writer.append_event, last-keyframe-time, pending timers). Do NOT collapse them — `stop()` spends non-trivial time joining threads and holding that on the per-event lock would starve capture callbacks. Capture-thread callbacks check `self._active` under `_lock` and bail if False, so late callbacks after `stop()` are silent no-ops.
- **FastAPI `Depends` pattern**: bind `Depends(...)` once at module level (`StateDep = Depends(get_recorder_state)`) and reuse it as each endpoint's default — ruff `B008` rejects inline `Depends(...)`, but FastAPI inspects the default for the `Depends` instance regardless. This is the canonical workaround.
- **Hermetic FastAPI testing**: drive the app via `httpx.AsyncClient(transport=ASGITransport(app=app))` (no real socket) and override dependencies via `app.dependency_overrides[get_recorder_state] = lambda: state`. Do NOT mutate module-level singletons in tests — they leak across tests because pytest-asyncio reuses the event loop.
- **HTTP path traversal testing**: `httpx.AsyncClient` normalises `..`/`.`/empty path segments BEFORE the request reaches the ASGI app, so HTTP-level traversal tests are unreliable. Unit-test the path-safety helper (e.g. `_safe_screenshot_path`) directly with raw filename strings instead of through the HTTP layer.
- **Storage root resolution**: `recorder.storage.default_trajectories_root()` resolves prod (`Trace`) vs dev (`Trace-dev`) profile via `TRACE_DEV_MODE=1`. Tests pin a `tmp_path` via `TRACE_PROFILE_DIR`. Any new module that needs to write to the per-user app data directory should reuse this helper rather than computing its own paths.
- **PermissionsError forwarding**: `PermissionsMissingError.error` is the structured `PermissionsError` TypedDict — return it verbatim as the 403 body via `JSONResponse(status_code=403, content=dict(exc.error))`. Synthesizer/Runner endpoints that need similar structured-error responses should follow the same shape rather than re-inventing.
- **IndexDB is a cache, not a source of truth**: `recorder.index_db.IndexDB` mirrors per-trajectory summary fields in SQLite for fast `GET /trajectories`. The filesystem (`metadata.json` + `events.jsonl`) remains authoritative; `IndexDB.reconcile(root)` rebuilds the cache from disk and is invoked automatically in `RecorderState.__init__` so a service restart heals drift without manual intervention. Writer failures against the index are silently suppressed via `contextlib.suppress(Exception)` — a cache outage must not break a live recording.
- **SessionFactory signature includes IndexDB**: `recorder.api.SessionFactory = Callable[[Path, IndexDB | None], RecordingSession]`. Any test that substitutes `session_factory=FakeSession` must have `FakeSession.__init__(self, root, index_db=None)`, and should forward the index_db to its `TrajectoryWriter` so `list_trajectories` assertions see the rows. Changing the factory again later is a breaking change across every test suite that mocks a session — think twice.
- **Where the index lives**: `<app_support>/index.db` (i.e., `default_trajectories_root().parent / "index.db"`), NOT inside the `trajectories/` directory.  `recorder.storage.default_index_db_path()` is the canonical resolver — reuse rather than hardcoding. Tests pin via the same `TRACE_PROFILE_DIR` override that controls `default_trajectories_root`.
- **duration_ms tests must tolerate sign**: `TrajectoryWriter.close()` rewrites `stopped_at` to `now()`, so a hermetic test whose `started_at` is a fixed future date will produce a negative `duration_ms`. Either assert on presence-not-sign (`is not None`), OR overwrite the index row post-close with deterministic values (that's what `FakeSession.stop()` in `test_api.py` does — then `list_trajectories` can assert `duration_ms == 1500`).
- **Fixtures are synthetic until a human re-records**: `fixtures/trajectories/<slug>/` is produced by `scripts/generate_synthetic_fixtures.py` (Ralph/CI path). Real recordings replace them during smoke testing per `scripts/regenerate_fixtures.sh`. Downstream tests that depend on fixtures must rely only on structural contracts (schema validity, per-type event counts) — never on the literal text of a `text_input` or the pixel contents of a screenshot. `tests/recorder/test_fixtures.py` is the reference structural check.

## 2026-04-22 - R-003

- `write_metadata(metadata)` validates via `recorder.schema.validate_metadata`, then writes atomically (`.tmp` → fsync → rename) and caches the dict so `close()` can finalise `stopped_at`.
- `append_event(event)` assigns `seq` if missing (or `None`), validates, JSON-encodes one line, flushes. A single `threading.Lock` guards seq assignment + file write so the CGEventTap background thread and the main-thread text aggregator can share a writer. - **Learnings for future iterations:**
  - `datetime.now(timezone.utc)` trips ruff `UP017` on Python 3.11+ — use `from datetime import UTC` and `datetime.now(UTC)`.
  - `tmp.replace(final)` on POSIX is atomic and works cross-device on the same filesystem; it's the right primitive for "write then swap into place".
  - `Path.with_suffix(".png.tmp")` is wrong for multi-dot suffixes (replaces the existing `.png`). Use `path.with_name(path.name + ".tmp")` to append a suffix safely.
  - When assigning `seq` before validating, copy the dict (`dict(event)`) so validation failures don't mutate the caller's input — this matters for the session orchestrator that may retry or re-route a rejected event.
  - The writer's `_lock` is coarse (guards both the counter and the file handle). That's fine for the expected event rate (well under 1k events/sec); if we ever need finer granularity, split into a `seq_lock` + `write_lock` — but beware of ordering: the on-disk order must match seq order for tests and for downstream Synthesizer consumption.
  - `close()` reading the metadata from disk as a fallback means that even if a test calls `close()` without ever calling `write_metadata()`, we don't crash — we just skip the `stopped_at` finalisation. The orchestrator should always call `write_metadata()` first, but the writer defends against the bad ordering.

## Codebase Patterns
- Cross-module contract consistency lives in `synthesizer.check_contracts` (wrapper: `scripts/check_contracts.sh`). It layers over `synthesizer.check_fixtures`: (1) JSON-Schema validity of every `contracts/*.schema.json` via `Draft202012Validator.check_schema` + explicit `$schema` dialect pin, (2) trajectory-fixture validity via `TrajectoryReader(traj_dir)` (reuses the reader's line-numbered error messages), (3) skill-fixture validity via `check_all_fixtures()` (schema + round-trip + markdown/meta cross-check), (4) `meta.trajectory_id → metadata.json.id` linkage inside `_check_trajectory_link`. Both scripts share the `CheckResult` API (`failures: list[str]`, `ok` property, `fail(path, reason)`) — downstream gate scripts should either extend this helper or mirror the same shape so failure messages stay uniform and LLM-actionable.
- `httpx.ASGITransport` does NOT stream response bodies — it buffers the full response before any bytes reach the client, so SSE endpoints served via sse-starlette appear to hang forever when consumed via `ac.stream("GET", ...)` + `aiter_lines`/`aiter_bytes` in tests. Test the generator directly (`api_module._sse_event_generator(session, fake_request)`) with a `_FakeRequest` stub exposing `is_disconnected()`; verify the route wiring separately (pin `SSE_PING_INTERVAL_SECONDS == 15` and walk `router.routes` to confirm the stream path exists). Full ASGI SSE integration belongs in a uvicorn-backed smoke test, not the unit suite.
- `_sse_event_generator` emits 3 events per state transition when the session is already past the first transition (state_change, draft_ready, question_ready all in ONE poll iteration). Tests that iterate the generator should `break` at the expected event count and then `request.disconnect()` so the next poll returns cleanly. The generator never exits on its own while the session is still in a non-terminal state — disconnect is the only clean termination hook aside from the session reaching COMPLETED/ERRORED/CANCELLED.
- Background tasks from `asyncio.create_task` must hold a strong reference to avoid GC mid-flight. `api.py` uses a module-level `_BACKGROUND_TASKS: set[asyncio.Task[None]]` plus a `add_done_callback(_BACKGROUND_TASKS.discard)` — downstream async stories (S-015 e2e) should use the same `_schedule_background` helper rather than calling `create_task` directly.
- uv workspace: dev extras on workspace members aren't picked up by `uv sync --extra dev` at the root; use `uv sync --all-extras --all-packages` (or run per-package) to install a workspace member's dev deps.
- Synthesizer package uses **src layout** (`services/synthesizer/src/synthesizer/`) with hatchling; declare `[tool.hatch.build.targets.wheel] packages = ["src/synthesizer"]` so the wheel finds it.
- `pytest` exits 5 on "no tests collected" — that's NOT zero. If a story's AC requires `pytest ... exits 0`, add at least one smoke test to the new suite.
- All synthesizer tests run with `TRACE_LLM_FAKE_MODE=1` + `ANTHROPIC_API_KEY=test-dummy-key` forced via an autouse fixture in `tests/synthesizer/conftest.py`, plus the `anthropic_mock` respx router is autouse-installed on every test so an unmocked call to `api.anthropic.com` raises instead of hitting the network.
- Per-service `pyproject.toml` declares its own `[tool.mypy] strict = true` so `uv run mypy --strict services/synthesizer` uses service-scoped config (not the root's). Root `[tool.mypy]` still covers `services` + `gateway` for repo-wide checks.
- `contracts/*.schema.json` is located at runtime by walking up from `Path(__file__)` until a directory containing `contracts/` is found; this keeps modules working from both the editable workspace install and a future published wheel without hard-coding a relative path.
- jsonschema's `FormatChecker` validates `uuid` out-of-the-box, but `date-time` requires the optional `rfc3339-validator` dep — schemas relying on strict date-time enforcement should combine `type: "string"` with a regex pattern or a post-validation Python check, not just the `format` keyword.
- `jsonschema` ships no PEP 561 type stubs; use `# type: ignore[import-untyped]` on the import line to keep `mypy --strict` green without pulling in `types-jsonschema`.
- When two branches of one function both assign `sorted(set_diff)` to the same variable name, mypy pins the variable's element type from the first assignment. Use distinct names per branch (e.g., `only_md_steps` vs `only_md_params`) to avoid spurious `incompatible type` errors.
- SKILL.md format (locked by S-003): sections in strict order `# Title` → description → `## Parameters` → `## Preconditions` → `## Steps` → `## Expected outcome` → optional `## Notes`. Parameter bullet shape: `` - `name` (type, required|optional[, default: value]) — description ``. Defaults: strings double-quoted, integers bare, booleans `true`/`false`. Empty bullet sections render `_None._`. Destructive steps prefixed with `⚠️ [DESTRUCTIVE]` exactly.
- Pydantic v2 `model_config = ConfigDict(frozen=True)` on `ParsedSkill`/`Step`/`Parameter` makes round-trip tests use bare `==` for equality and also prevents accidental mutation by downstream stages.
- The parameter-reference regex `(?<!\\)(?<!\{)\{NAME\}(?!\})` plus pre-stripping fenced/inline code is the approved pattern for finding `{name}` refs — mirrors S-002 schema cross-check but is stricter (handles escaped `\{`, `{{double}}`, inline backticks). Use `extract_parameter_refs` from `skill_doc` rather than re-rolling the regex in future stories.
- The trajectory schema (`contracts/trajectory.schema.json`) requires every event to carry `seq` (integer ≥ 1, unique per trajectory), and supports `click` and `app_switch` event kinds plus an optional `target: {label, role, bundle_id}` sub-object on clicks. Downstream stories (S-005 preprocessor, S-008 destructive matcher) read click `target.label` — populate it on every click event in fixtures.

- `synthesizer.destructive_matcher.apply_destructive_matcher` uses **label-in-step-text binding**, not positional index-to-click pairing: for each click with a destructive label, find the earliest un-bound step whose text case-insensitively contains the label substring. - `synthesizer.slug`: canonical slug shape is `^[a-z][a-z0-9_]{2,39}$` (constants `SLUG_PATTERN`, `SLUG_REGEX`, `SLUG_MIN_LEN=3`, `SLUG_MAX_LEN=40`). `slugify(name)` pipeline: NFKD transliterate → lowercase → collapse any `[^a-z0-9_]+` run into single `_` → strip leading non-letters (digits, underscores, leftover junk) → rstrip `_` → truncate to 40, rstrip `_` again → error if <3 chars. `resolve_unique_slug(base, dir)` only uses numeric suffixes (`base_2`, `base_3`, ...) — the `_v2`/`_copy`/`_new` naming ambiguity is disallowed by design. `validate_user_slug(input, dir) -> (bool, reason)` returns human-readable rejection messages suitable for surfacing to the user in the Q&A UI. Downstream (S-010 writer, S-011 session approve, S-013 API `/approve` endpoint) should always call `resolve_unique_slug` before a disk write and prefer `validate_user_slug` before accepting a user-provided override.
- `synthesizer.writer.SkillWriter.write(parsed, meta, reader, skills_root, *, cost_total_usd=0.0) -> WrittenSkill` is the ONLY allowed path for creating files under `skills/` or touching `index.db`. Any failure before or during commit rolls back the entire skill directory via `shutil.rmtree(..., ignore_errors=True)` — the invariant "skill is either fully registered or fully absent" is enforced, not hoped for.

- Preview screenshot selection (`synthesizer.writer._pick_preview_indices(n, cap)`): when `n <= cap`, returns `[0..n-1]`; otherwise spread across `[0, n-1]` using `round(i * (n-1) / (cap-1))` for `i in range(cap)` and dedupe while preserving order. Python's `round` uses banker's rounding so `n=6, cap=5` → `[0, 1, 2, 4, 5]` and `n=100, cap=5` → `[0, 25, 50, 74, 99]` — tests pin these exact values. Screenshots are copied (not symlinked) via `shutil.copy2`; the AC is explicit so the skill dir is self-contained even if the trajectory is later deleted.

The `os.fsync` is imported as `synthesizer.writer.os.fsync` for monkeypatch reachability in crash-simulation tests — reference it as `"synthesizer.writer.os.fsync"` in `monkeypatch.setattr` rather than patching the global `os` module.
- `synthesizer.session.SynthesisSession` is the state-machine orchestrator: states are a `StrEnum` (`generating_draft`, `awaiting_answer`, `generating_revision`, `awaiting_approval`, `saving`, `completed`, `cancelled`, `errored`); transitions go through `_transition(target)` which consults the declarative `_LEGAL_TRANSITIONS` dict and raises `IllegalStateTransition` on any illegal move. The constructor is intentionally non-blocking (no thread spawn) — `start_draft()` is explicit so the HTTP layer (S-013) can wrap it in `asyncio.to_thread` while unit tests run it inline. `draft_fn` and `revise_fn` are injected callables (Protocols `DraftFn` / `ReviseFn`) so S-012 can plug in `generate_revision` without a circular dep, and tests can script behaviour deterministically.

Persistence happens AFTER the state transition so `draft.json` always reflects the CURRENT resting state (`awaiting_answer`/`awaiting_approval`), never an intermediate (`generating_draft`). v1 does NOT rehydrate sessions from disk on service restart — documented PRD limitation.
- Session approval slug handling: when the caller passes `slug=None`, the LLM-suggested slug in `draft.meta["slug"]` is piped through `resolve_unique_slug` (silent numeric suffix on collision); when the caller passes an explicit `slug="..."`, it goes through `validate_user_slug` and a collision raises `SlugError` rather than renaming. Matches the PRD's user-vs-LLM asymmetry: users who type a slug expect to see their exact input or an error, not a silent rename.

## 2026-04-23 - S-011 Synthesis session state machine
- Added `services/synthesizer/src/synthesizer/session.py` (~420 lines) exposing `SynthesisSession`, `SessionStore`, `SessionState` (StrEnum), `IllegalStateTransition`, `DraftFn` / `ReviseFn` Protocols, `DEFAULT_SESSION_TTL_SECONDS=1800`, `session_dir()`, `sessions_root()`, plus the `_LEGAL_TRANSITIONS` transition table and `TERMINAL_STATES` frozenset.
- `SynthesisSession` constructor is cheap: allocates a session_id (uuid4 hex), stores reader/client/writer references, sets state=GENERATING_DRAFT. No threading — `start_draft()` is explicit so tests can exercise the illegal-transition case (answer_question while state=generating_draft) without racing a background thread.
- State-machine core: `_transition(target)` consults `_LEGAL_TRANSITIONS` and raises `IllegalStateTransition` on illegal moves. Legal flow: GENERATING_DRAFT → AWAITING_ANSWER (or AWAITING_APPROVAL if no questions) → GENERATING_REVISION → AWAITING_ANSWER/AWAITING_APPROVAL → SAVING → COMPLETED. Any non-terminal state can transition to CANCELLED or ERRORED. Terminal states have no legal outgoing transitions (approve-after-completion, cancel-after-completion both raise).
- Draft phase: `start_draft()` calls `preprocess_trajectory(reader)` then `draft_fn(preprocessed, client, *, reader)` matching `generate_draft`'s signature. Catches `DraftGenerationError` and unexpected exceptions, both route to `_fail(msg)` which sets `error` and transitions to ERRORED. On success, stores the draft, seeds `questions` from `draft.questions`, sets `costs_so_far_usd = draft.total_cost_usd`, transitions to the appropriate next state, then calls `_persist_draft()`.
- Q&A phase: `answer_question(question_id, text)` validates state=AWAITING_ANSWER, non-empty text (raises ValueError on blank), known question_id (raises ValueError on unknown). Transitions to GENERATING_REVISION, calls `revise_fn(current_draft=..., question=..., answer=..., client=..., reader=...)`, catches errors into ERRORED. On success, appends to `answered`, updates draft/questions/costs, then transitions to AWAITING_ANSWER if `next_question() is not None` else AWAITING_APPROVAL. `next_question()` returns the first question whose id isn't in `answered`.
- Approve phase: `approve(slug=None)` validates state=AWAITING_APPROVAL, transitions to SAVING, resolves slug (`resolve_unique_slug` for None, `validate_user_slug` for user-provided), calls `SkillWriter.write(...)`, transitions to COMPLETED and cleans up persistence. `SlugError` from user-provided collision transitions to ERRORED and re-raises so the API layer can return 409. Any other write failure also transitions to ERRORED and re-raises.
- Cancel: `cancel()` transitions any non-terminal state → CANCELLED, cleans up `synthesis_sessions/<sid>/` via `shutil.rmtree(..., ignore_errors=True)`. Calling cancel on a terminal state raises `IllegalStateTransition`.
- Persistence: `_persist_draft()` writes `<trace_data_dir>/synthesis_sessions/<session_id>/draft.json` with {session_id, trajectory_id, state, markdown, meta, questions, answered, costs_so_far_usd}. Called AFTER state transitions so the file reflects the resting state, not an intermediate. `snapshot()` returns a UI-facing dict with the same shape plus `next_question` and `error`.
- `SessionStore`: in-memory dict keyed by session_id. `add`/`get`/`remove`/`__contains__`/`__len__`/`ids()` + `evict_stale()` which prunes both terminal sessions and non-terminal sessions older than `ttl_seconds` (default 1800s = 30min). Injectable `clock: Callable[[], float]` for fake-time tests. `cancel_cleanup()`/`cleanup_cancelled()` are hooks for the S-013 async cleanup loop.
- Added `tests/synthesizer/test_session.py` with 23 tests: full happy-path (3 questions → approve → WrittenSkill), no-questions draft skips straight to AWAITING_APPROVAL, illegal transitions (answer during generating_draft, approve before draft, start_draft after completion, cancel from terminal), cancel from each non-terminal state (generating_draft, awaiting_answer with persistence cleanup, awaiting_approval with no skill written), draft + revision error paths, approve with colliding LLM slug (renames to `_2`), approve with colliding user slug (raises SlugError + ERRORED state), approve with custom user slug (honored verbatim), empty/whitespace answer raises ValueError without changing state, unknown question_id raises, draft persistence file written + updated after revision, SessionStore add/get/remove/duplicate-rejection, evict_stale past TTL with fake clock, evict_stale prunes terminal sessions, cleanup_cancelled flag, snapshot shape progression with state.

- **Learnings for future iterations:**
  - `ruff UP042` flags `class Foo(str, Enum)` and suggests `class Foo(StrEnum)` — Python 3.11's `enum.StrEnum` is cleaner and members still behave like strings (so `"generating_draft" == SessionState.GENERATING_DRAFT` holds). Use StrEnum for any new string-valued enum in this codebase.

- **Learnings for future iterations:**
  - `unicodedata.normalize("NFKD", s).encode("ascii", "ignore").decode("ascii")` is the stdlib pattern for transliteration — no Unidecode dep needed. It splits combining diacritics off their base letters, then the ASCII-ignore encode drops the marks (and every non-ASCII codepoint, including emoji, CJK, etc.). Downstream code that needs transliteration should reuse this idiom — it's about 5 lines, not worth a helper.
  - `str.rstrip("_")` after truncation is mandatory — slicing `s[:40]` can leave a trailing `_` if position 40 happened to be a separator, and the resulting slug would fail `SLUG_REGEX`. The slugify tests hit this boundary explicitly (`"a"*39 + " " + "b"*10`).
  - The pattern `^[a-z][a-z0-9_]{2,39}$` bounds TOTAL length to 3–40 (first char + 2–39 more). A common off-by-one would be to think "2 to 39 more" means total 2–39. The tests at the min/max boundaries (`"abc"` and `"a" + "b"*39`) exist to pin this down.
  - `SlugError` subclasses `ValueError` so callers that catch `ValueError` broadly still see it; keep it as a specific subclass so new call sites can tighten up.
  - Ruff's `RUF059` flags unused unpacked tuple members (`ok, reason = ...` when only `ok` is asserted). The fix is to either prefix with `_` or actually use the variable; the test suite now asserts on the reason text wherever it's unpacked, which doubles as a sanity check on the user-facing message quality.

  - Module-level factory hooks (`_BUDGET_MONITOR_FACTORY: Callable[[], BudgetMonitor] = BudgetMonitor`) are the established pattern for DI in `api.py` — matches the existing `_DRAFT_FN`/`_REVISE_FN`/`_STORE` naming. Tests inject via `monkeypatch.setattr(api_module, "_BUDGET_MONITOR_FACTORY", _factory)`. Do not pass monitors through request state or FastAPI Depends — the module-level swap keeps the endpoint signature clean and matches the existing conventions.
  - `SynthesisSession`'s `_enforce_budget()` is called after `costs_so_far_usd` is updated, BEFORE the next state transition. Calling it before the cost update would race (costs hasn't accumulated yet); calling after the transition would let the state leak forward one step past the cap. This order is load-bearing — if future stories add new cost-accruing state transitions (e.g., a separate "polish" phase), insert `_enforce_budget()` calls at the same post-update / pre-transition point.
  - The daily-cost check in `/synthesize/start` runs BEFORE `trajectory_dir.is_dir()` — a client hitting a not-yet-existing trajectory with the daily cap exceeded gets a 429, not a 404. This is deliberate: the cap is an ambient constraint on the system, while the 404 is conditional on the caller's input. Error priority matters for UI behavior (a 429 should suggest "try tomorrow", a 404 should suggest "record first").
  - The monitor writes warnings to stderr via bare `print(..., file=sys.stderr)` AND `LOGGER.warning(...)`. This belt-and-suspenders pattern ensures the warning reaches the user even when logging is configured above WARNING level (common in daemon contexts where `logging.basicConfig(level=ERROR)` is set to quiet noisy deps). Reusable for any future guardrail where "user sees this" matters more than "logs capture this".
