## Codebase Patterns
- uv workspace: dev extras on workspace members aren't picked up by `uv sync --extra dev` at the root; use `uv sync --all-extras --all-packages` (or run per-package) to install a workspace member's dev deps.
- Synthesizer package uses **src layout** (`services/synthesizer/src/synthesizer/`) with hatchling; declare `[tool.hatch.build.targets.wheel] packages = ["src/synthesizer"]` so the wheel finds it.
- `pytest` exits 5 on "no tests collected" — that's NOT zero. If a story's AC requires `pytest ... exits 0`, add at least one smoke test to the new suite.
- All synthesizer tests run with `TRACE_LLM_FAKE_MODE=1` + `ANTHROPIC_API_KEY=test-dummy-key` forced via an autouse fixture in `tests/synthesizer/conftest.py`, plus the `anthropic_mock` respx router is autouse-installed on every test so an unmocked call to `api.anthropic.com` raises instead of hitting the network.
- Per-service `pyproject.toml` declares its own `[tool.mypy] strict = true` so `uv run mypy --strict services/synthesizer` uses service-scoped config (not the root's). Root `[tool.mypy]` still covers `services` + `gateway` for repo-wide checks.
- `contracts/*.schema.json` is located at runtime by walking up from `Path(__file__)` until a directory containing `contracts/` is found; this keeps modules working from both the editable workspace install and a future published wheel without hard-coding a relative path.
- jsonschema's `FormatChecker` validates `uuid` out-of-the-box, but `date-time` requires the optional `rfc3339-validator` dep — schemas relying on strict date-time enforcement should combine `type: "string"` with a regex pattern or a post-validation Python check, not just the `format` keyword.
- `jsonschema` ships no PEP 561 type stubs; use `# type: ignore[import-untyped]` on the import line to keep `mypy --strict` green without pulling in `types-jsonschema`.
- When two branches of one function both assign `sorted(set_diff)` to the same variable name, mypy pins the variable's element type from the first assignment. Use distinct names per branch (e.g., `only_md_steps` vs `only_md_params`) to avoid spurious `incompatible type` errors.
- SKILL.md format (locked by S-003): sections in strict order `# Title` → description → `## Parameters` → `## Preconditions` → `## Steps` → `## Expected outcome` → optional `## Notes`. Parameter bullet shape: `` - `name` (type, required|optional[, default: value]) — description ``. Defaults: strings double-quoted, integers bare, booleans `true`/`false`. Empty bullet sections render `_None._`. Destructive steps prefixed with `⚠️ [DESTRUCTIVE]` exactly.
- Pydantic v2 `model_config = ConfigDict(frozen=True)` on `ParsedSkill`/`Step`/`Parameter` makes round-trip tests use bare `==` for equality and also prevents accidental mutation by downstream stages.
- The parameter-reference regex `(?<!\\)(?<!\{)\{NAME\}(?!\})` plus pre-stripping fenced/inline code is the approved pattern for finding `{name}` refs — mirrors S-002 schema cross-check but is stricter (handles escaped `\{`, `{{double}}`, inline backticks). Use `extract_parameter_refs` from `skill_doc` rather than re-rolling the regex in future stories.
- The trajectory schema (`contracts/trajectory.schema.json`) requires every event to carry `seq` (integer ≥ 1, unique per trajectory), and supports `click` and `app_switch` event kinds plus an optional `target: {label, role, bundle_id}` sub-object on clicks. Downstream stories (S-005 preprocessor, S-008 destructive matcher) read click `target.label` — populate it on every click event in fixtures.
- Reference-fixture trajectories live at `fixtures/trajectories/<slug>/` with `metadata.json`, `events.jsonl`, and `screenshots/0001.png` (a 67-byte 1×1 grayscale PNG). Tests that mutate a fixture MUST `shutil.copytree` it into `tmp_path` first — the committed fixtures are ground truth for S-016 and other downstream tests.
- `TrajectoryReader` validates `metadata.json` against `definitions.metadata` and each `events.jsonl` line against `definitions.event` using two separate `Draft202012Validator` instances, cached via `lru_cache`. Error messages always include the line number AND the event's `seq` so a malformed fixture is trivial to locate.
- Event objects use `ConfigDict(extra='allow', frozen=True)` — same pattern as `ParsedSkill`. Unknown keys flow through for forward-compat, and frozen events are safe to cache / hash / compare in downstream stages.
- Events are sorted by `seq` on load, so `iter_events()` / `iter_events_by_type()` always yield in logical order regardless of on-disk line order. Duplicate `seq` values raise on construction (would make `get_screenshot_path` ambiguous).
