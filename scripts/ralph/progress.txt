## Codebase Patterns
- `services/recorder` uses a **src layout** (`services/recorder/src/recorder/‚Ä¶`); future files (schema.py, writer.py, session.py, etc.) go under `src/recorder/`.
- The recorder package is installed as a uv workspace member (`trace-recorder`) and should be imported as `from recorder.xyz import ‚Ä¶`, NOT `services.recorder.recorder.xyz`.
- Root `pyproject.toml` owns shared tooling config (ruff/mypy/pytest); per-service `pyproject.toml` only declares that service's runtime deps + hatchling build.
- Ruff's isort needs `known-first-party = ["recorder", "synthesizer", "runner", "gateway"]` or it misgroups the workspace imports in gateway/main.py.
- PyObjC wheels only exist for macOS ‚Äî deps are marked `; sys_platform == 'darwin'` so `uv sync` still works if Ralph ever runs on non-darwin.
- `mypy --strict` has PyObjC module overrides in root pyproject (`Cocoa.*`, `Quartz.*`, `ApplicationServices.*`, `AppKit.*`, `Foundation.*`, `objc.*`); add new PyObjC module imports here, not in each file.
- Quality-gate commands for the recorder module are:
  - `uv run ruff check services/recorder`
  - `uv run mypy --strict services/recorder`
  - `uv run pytest tests/recorder/`
- Pre-existing gateway/integration test failures (`ModuleNotFoundError: gateway`) are NOT caused by R-001 and are outside this branch's scope ‚Äî a future `feat/integration` story will install gateway as a proper package.
- **Schema validation**: `recorder.schema` exposes `validate_event(event)` / `validate_metadata(meta)` backed by `jsonschema`'s Draft 2020-12 validator (with `FORMAT_CHECKER` enabled for `uuid`/`date-time`). Future writer/session code should call these before writing JSON to disk so bad events never land in `events.jsonl`.
- **Schema file lookup**: `recorder.schema.find_schema_path()` walks up from `__file__` to find `contracts/trajectory.schema.json`. Works for the uv-workspace editable install. If the contracts ever move or get bundled into the wheel, update this function ‚Äî do NOT hardcode a path.
- **Local `$ref` only**: the `load_event_schema()` / `load_metadata_schema()` wrappers build standalone schemas with a `#/$defs/...` fragment ref (no `$id`), which keeps `jsonschema` from trying to HTTP-fetch the schema's `$id` URL at validate time. If you add new top-level sub-schemas for Synthesizer/Runner to reuse, follow the same pattern.
- **TrajectoryWriter ownership**: `recorder.writer.TrajectoryWriter` is the only module allowed to mutate the on-disk trajectory directory. All writes go through its lock; session/orchestrator code must not write `events.jsonl` / `metadata.json` / `screenshots/*` directly. It validates via `recorder.schema.validate_{event,metadata}` BEFORE any disk write, so bad data never leaks to disk. The constructor creates `<root>/<uuid>/` + `screenshots/` with perms `0700` and the writer is usable as a context manager (`__exit__` calls `close()`).
- **Atomic JSON writes**: `_atomic_write_json(path, data)` writes to `<path>.tmp`, fsyncs, and renames. Use it for any metadata-style file. `events.jsonl` is append-flush (no fsync per line ‚Äî documented v1 tradeoff).
- **Datetime import style**: use `from datetime import UTC, datetime` (Python 3.11+). Ruff's `UP017` flags `timezone.utc` ‚Äî don't revive it. ISO strings are produced with `datetime.now(UTC).isoformat(timespec="milliseconds")` which yields `+00:00` suffixes; the schema's `format: date-time` accepts both `+00:00` and `Z`.
- **Ruff SIM105**: prefer `with contextlib.suppress(OSError): os.fsync(...)` over `try/except/pass` for best-effort fsync on filesystems that don't support it (tmpfs in pytest tmp_path).
- **PyObjC mocking in tests**: for any module that does `from ApplicationServices import X` / `from Quartz import Y` inside a function, tests stub the framework by installing a `types.ModuleType("Quartz")` / `types.ModuleType("ApplicationServices")` into `sys.modules` via `monkeypatch.setitem`. The import inside the tested function then resolves to the stub. For ImportError coverage, install a stub that is missing the expected attributes ‚Äî `from X import missing_attr` raises ImportError, which the module under test should catch.
- **Permission-check pattern**: `recorder.permissions` keeps each macOS framework call in its own thin `_check_*` helper, and `check_permissions()` / `get_missing_permissions_error()` simply compose them. Tests patch the helpers for combination coverage and patch the PyObjC symbols inside `sys.modules` for helper-level coverage. Use the same pattern for any future PyObjC-backed module (event_tap, ax_resolver, focus_tracker, screenshot) to keep the suite hermetic on Ralph's sandbox.
- **`@pytest.mark.macos` is opt-in only**: `tests/recorder/conftest.py` auto-skips every test tagged `macos` unless `TRACE_RUN_MACOS_TESTS=1` is set. Use this marker for ANY test that needs real PyObjC framework behaviour (CGEventTap, AXUIElement, CGWindowListCreateImage, etc.). Hermetic unit coverage should still be provided alongside via `sys.modules`-stubbed Quartz/ApplicationServices so the default run stays green.
- **Splitting PyObjC callback from normaliser**: for modules driven by a C-callback (CGEventTap), put the pure event‚Üídict conversion in a module-level function that takes the framework module as a parameter (`_normalise_event(code, event, quartz)`), and expose a thin `handle_cg_event(code, event)` method on the class that the tap callback delegates to. Tests exercise the normaliser + re-enable logic by calling `handle_cg_event` with fake events ‚Äî no run-loop or thread needed. `event_tap.py` is the reference implementation.
- **Re-enabling a disabled tap**: `CGEventTapEnable(tap, True)` re-arms a tap the OS disabled via `kCGEventTapDisabledByTimeout` / `kCGEventTapDisabledByUserInput`. Both constants appear as unsigned `uint32` (0xFFFFFFFE / 0xFFFFFFFF) in headers but PyObjC sometimes surfaces them as signed ints (-2 / -1); match both forms.
- **Bounded-time framework calls**: when a PyObjC call could block indefinitely (AX queries, inter-app RPC), wrap it with a daemon `threading.Thread` and a `threading.Event` set in a `finally`. The caller `event.wait(timeout)` ‚Äî on timeout the worker thread just keeps living as a daemon. Do not rely on `Thread.join(timeout)` alone to tell "done" from "still running"; always use a separate event sentinel. See `ax_resolver.resolve_element_at` for the reference.
- **PyObjC AX value shapes**: AX position/size attributes come back as one of three shapes depending on macOS/PyObjC version ‚Äî native 2-tuple, bridged object with `.x/.y` or `.width/.height`, or raw `AXValueRef` that must be unwrapped via `AXValueGetValue(value, kAXValueCG{Point,Size}Type, None) -> (success, unpacked)`. Any new AX-consuming code should tolerate all three (see `_unpack_ax_value` in ax_resolver.py).
- **NSWorkspace subscription pattern**: prefer `addObserverForName:object:queue:usingBlock:` (a Python callable passed as block) over the NSObject-subclass selector pattern ‚Äî it keeps the handler method on the Python class and avoids `@objc.selector` machinery. The block returns an opaque observer handle that must be passed back to `removeObserver:` on stop. Tests stub `AppKit.NSWorkspace.sharedWorkspace().notificationCenter()` with a class that captures `(name, block)` in a list so we can invoke the block synchronously with a fake notification. See `focus_tracker._subscribe_workspace` for the reference.
- **NSDictionary dual-access**: `userInfo()` on an `NSNotification` is bridged to a Python dict in most PyObjC builds but not all. Accept both by trying `.get(key)` first (callable on a bridged dict) and falling back to `objectForKey_(key)` on an ObjC-only stub. Same pattern applies to any NSDictionary the recorder reads out of a framework callback.
- **Idle-flush timer pattern**: for any module that aggregates into a buffer until a "nothing happened for N seconds" signal, use a `threading.RLock` around the state + a `threading.Timer` that is cancelled-and-replaced on every ingest event. The RLock (not a plain Lock) is required because the Timer's callback may take the lock while a caller-driven `flush()` is mid-flight. The Timer callback should also check an `is_stopped` flag so a late-firing cancelled timer is a silent no-op. `text_aggregator.py` is the reference.
- **Session-layer translation of event_tap output**: `recorder.event_tap` emits modifier names in PyObjC framework style (`"command"`, `"control"`, `"option"`, ‚Ä¶). The trajectory schema's `modifier` enum uses short forms (`"cmd"`, `"ctrl"`, `"opt"`, ‚Ä¶). Translation happens in the session orchestrator (R-010) when shaping a schema-valid `keypress` event. Downstream consumers of event_tap (text_aggregator, session) must agree on which name style they expect ‚Äî currently everything inside the recorder uses the long PyObjC names; ONLY the session's final schema-event builder maps to the short enum.
- **Stateless policy modules**: `recorder.keyframe_policy.KeyframePolicy` is stateless ‚Äî the session passes `(event_type, seconds_since_last_keyframe)` each call and the policy answers yes/no + reason. The session owns the "time since last keyframe" state and the periodic timer. Prefer this shape for any future rule-based gate (e.g. scroll-debounce) ‚Äî stateless is easier to test, thread-safe by construction, and doesn't couple the rule to a particular timer impl.
- **Class-level state in PyObjC test doubles**: when emulating the `cls.alloc().init‚Ä¶_(‚Ä¶)` two-step idiom, do NOT use mutable class attributes at the base class (RUF012). Either put state on the instance, or declare `ClassVar[...]` explicitly, or create a fresh subclass per test via `type("Name", (Base,), {...})` ‚Äî the screenshot stub uses the subclass trick so each test's `rep_calls` list is isolated.
- **PNG validation without Pillow**: the `recorder.screenshot` module exposes `PNG_MAGIC = b"\\x89PNG\\r\\n\\x1a\\n"` and rejects bytes that don't start with it. Tests synthesize a minimal valid PNG by hand (magic + IHDR + one compressed IDAT + IEND, with `zlib.crc32` for the per-chunk CRC) ‚Äî avoid adding Pillow just to verify PNG shape. Dimension parsing uses `struct.unpack(">II", png[16:24])` once you've asserted `png[12:16] == b"IHDR"`.
- **RecordingSession owns `seq`**: `recorder.session.RecordingSession` is the single place where `seq` values are minted for a running trajectory. `TrajectoryWriter` still has its own auto-seq counter (belt-and-suspenders), but every normal emit path passes a pre-assigned seq under the session's `_lock`. Any future capture path that does NOT go through the session (e.g. an HTTP admin endpoint that injects events) must either acquire the same lock or accept interleaved seqs.
- **Session dependency injection**: every macOS-dependent piece of `RecordingSession` is injected via a constructor kwarg (`event_tap_factory`, `focus_tracker_factory`, `text_aggregator_factory`, `permissions_check`, `ax_resolver`, `screenshot_capturer`, `display_info_provider`, `keyframe_policy`). Hermetic tests pass fakes; real use passes nothing and defaults fall back to the real modules. Keep this pattern for any future orchestrator ‚Äî it is the only way tests can run in Ralph's sandbox without PyObjC.
- **Dual-lock orchestrator pattern**: `_start_stop_lock` serialises `start()`/`stop()` bookkeeping; `_lock` guards per-event state (seq, writer.append_event, last-keyframe-time, pending timers). Do NOT collapse them ‚Äî `stop()` spends non-trivial time joining threads and holding that on the per-event lock would starve capture callbacks. Capture-thread callbacks check `self._active` under `_lock` and bail if False, so late callbacks after `stop()` are silent no-ops.
- **FastAPI `Depends` pattern**: bind `Depends(...)` once at module level (`StateDep = Depends(get_recorder_state)`) and reuse it as each endpoint's default ‚Äî ruff `B008` rejects inline `Depends(...)`, but FastAPI inspects the default for the `Depends` instance regardless. This is the canonical workaround.
- **Hermetic FastAPI testing**: drive the app via `httpx.AsyncClient(transport=ASGITransport(app=app))` (no real socket) and override dependencies via `app.dependency_overrides[get_recorder_state] = lambda: state`. Do NOT mutate module-level singletons in tests ‚Äî they leak across tests because pytest-asyncio reuses the event loop.
- **HTTP path traversal testing**: `httpx.AsyncClient` normalises `..`/`.`/empty path segments BEFORE the request reaches the ASGI app, so HTTP-level traversal tests are unreliable. Unit-test the path-safety helper (e.g. `_safe_screenshot_path`) directly with raw filename strings instead of through the HTTP layer.
- **Storage root resolution**: `recorder.storage.default_trajectories_root()` resolves prod (`Trace`) vs dev (`Trace-dev`) profile via `TRACE_DEV_MODE=1`. Tests pin a `tmp_path` via `TRACE_PROFILE_DIR`. Any new module that needs to write to the per-user app data directory should reuse this helper rather than computing its own paths.
- **PermissionsError forwarding**: `PermissionsMissingError.error` is the structured `PermissionsError` TypedDict ‚Äî return it verbatim as the 403 body via `JSONResponse(status_code=403, content=dict(exc.error))`. Synthesizer/Runner endpoints that need similar structured-error responses should follow the same shape rather than re-inventing.
- **IndexDB is a cache, not a source of truth**: `recorder.index_db.IndexDB` mirrors per-trajectory summary fields in SQLite for fast `GET /trajectories`. The filesystem (`metadata.json` + `events.jsonl`) remains authoritative; `IndexDB.reconcile(root)` rebuilds the cache from disk and is invoked automatically in `RecorderState.__init__` so a service restart heals drift without manual intervention. Writer failures against the index are silently suppressed via `contextlib.suppress(Exception)` ‚Äî a cache outage must not break a live recording.
- **SessionFactory signature includes IndexDB**: `recorder.api.SessionFactory = Callable[[Path, IndexDB | None], RecordingSession]`. Any test that substitutes `session_factory=FakeSession` must have `FakeSession.__init__(self, root, index_db=None)`, and should forward the index_db to its `TrajectoryWriter` so `list_trajectories` assertions see the rows. Changing the factory again later is a breaking change across every test suite that mocks a session ‚Äî think twice.
- **Where the index lives**: `<app_support>/index.db` (i.e., `default_trajectories_root().parent / "index.db"`), NOT inside the `trajectories/` directory.  `recorder.storage.default_index_db_path()` is the canonical resolver ‚Äî reuse rather than hardcoding. Tests pin via the same `TRACE_PROFILE_DIR` override that controls `default_trajectories_root`.
- **duration_ms tests must tolerate sign**: `TrajectoryWriter.close()` rewrites `stopped_at` to `now()`, so a hermetic test whose `started_at` is a fixed future date will produce a negative `duration_ms`. Either assert on presence-not-sign (`is not None`), OR overwrite the index row post-close with deterministic values (that's what `FakeSession.stop()` in `test_api.py` does ‚Äî then `list_trajectories` can assert `duration_ms == 1500`).
- **Fixtures are synthetic until a human re-records**: `fixtures/trajectories/<slug>/` is produced by `scripts/generate_synthetic_fixtures.py` (Ralph/CI path). Real recordings replace them during smoke testing per `scripts/regenerate_fixtures.sh`. Downstream tests that depend on fixtures must rely only on structural contracts (schema validity, per-type event counts) ‚Äî never on the literal text of a `text_input` or the pixel contents of a screenshot. `tests/recorder/test_fixtures.py` is the reference structural check.

- **Learnings for future iterations:**
  - `datetime.now(timezone.utc)` trips ruff `UP017` on Python 3.11+ ‚Äî use `from datetime import UTC` and `datetime.now(UTC)`.
  - `tmp.replace(final)` on POSIX is atomic and works cross-device on the same filesystem; it's the right primitive for "write then swap into place".
  - `Path.with_suffix(".png.tmp")` is wrong for multi-dot suffixes (replaces the existing `.png`). Use `path.with_name(path.name + ".tmp")` to append a suffix safely.

  - `close()` reading the metadata from disk as a fallback means that even if a test calls `close()` without ever calling `write_metadata()`, we don't crash ‚Äî we just skip the `stopped_at` finalisation. The orchestrator should always call `write_metadata()` first, but the writer defends against the bad ordering.

## Codebase Patterns
- Runner uses a **src layout** (`services/runner/src/runner/`). Tests import `from runner.xxx`, resolved via `pythonpath = ["services/runner/src"]` in the root `pyproject.toml` pytest config. Do NOT add a nested `services/runner/runner/` package; that was the old flat layout and has been removed.
- The root pyproject owns the shared `ruff`, `mypy`, and `pytest` config. The runner's own `pyproject.toml` only adds a mypy `files = ["src"]` override so `uv run mypy --strict services/runner` targets the runner package directly.
- Safety gate: every live-mode code path must call `runner.safety.require_live_mode()` in `__init__`. The gate is strict string equality against `TRACE_ALLOW_LIVE="1"` ‚Äî `"true"`, `"yes"`, `"0"`, etc. all disable live mode. `tests/runner/conftest.py` autouses a fixture that force-unsets the flag before every test; opt in with the `live_mode_allowed` fixture.
- Tests mock Anthropic via `respx` (see `anthropic_mock` fixture) ‚Äî never hit the real API.
- Gateway (`gateway/main.py`) imports the runner as `from runner.api import router` (installed-package style via the pytest pythonpath). Synthesizer/recorder are still on the flat layout and import via `from services.{name}.{name}.api` ‚Äî those packages will switch to src layout when their respective branches reach X-001-equivalent stories.
- JSON schemas in `contracts/` are the on-disk contract; the runner pairs each with a pydantic model in `runner.schema` (e.g. `RunMetadata`) for typed in-memory use. Both must agree ‚Äî `extra="forbid"` on the model mirrors `additionalProperties: false` on the schema. Round-tripping uses `model_dump(mode="json", exclude_unset=True)` so unset optional fields do not leak back as nulls.
- Pydantic v2 serializes UTC datetimes with a trailing `Z` (not `+00:00`) via `mode="json"`. Tests that assert round-trip identity must store the canonical `Z` form in fixtures.
- `types-jsonschema` is in the shared dev extras so mypy --strict accepts `jsonschema` imports without module-level overrides.

Python's `round` uses banker's rounding so `n=6, cap=5` ‚Üí `[0, 1, 2, 4, 5]` and `n=100, cap=5` ‚Üí `[0, 25, 50, 74, 99]` ‚Äî tests pin these exact values. Screenshots are copied (not symlinked) via `shutil.copy2`; the AC is explicit so the skill dir is self-contained even if the trajectory is later deleted.
- Atomic text write (`synthesizer.writer.SkillWriter._atomic_write_text`): `open(".tmp")` ‚Üí `write` ‚Üí `flush` ‚Üí `os.fsync` ‚Üí `os.rename`. The `os.fsync` is imported as `synthesizer.writer.os.fsync` for monkeypatch reachability in crash-simulation tests ‚Äî reference it as `"synthesizer.writer.os.fsync"` in `monkeypatch.setattr` rather than patching the global `os` module.
- `synthesizer.session.SynthesisSession` is the state-machine orchestrator: states are a `StrEnum` (`generating_draft`, `awaiting_answer`, `generating_revision`, `awaiting_approval`, `saving`, `completed`, `cancelled`, `errored`); transitions go through `_transition(target)` which consults the declarative `_LEGAL_TRANSITIONS` dict and raises `IllegalStateTransition` on any illegal move. The constructor is intentionally non-blocking (no thread spawn) ‚Äî `start_draft()` is explicit so the HTTP layer (S-013) can wrap it in `asyncio.to_thread` while unit tests run it inline. `draft_fn` and `revise_fn` are injected callables (Protocols `DraftFn` / `ReviseFn`) so S-012 can plug in `generate_revision` without a circular dep, and tests can script behaviour deterministically.
- Session persistence: every successful draft + revision writes `<trace_data_dir>/synthesis_sessions/<session_id>/draft.json`; `cancel()` and `approve() ‚Üí COMPLETED` both `shutil.rmtree` the dir. Persistence happens AFTER the state transition so `draft.json` always reflects the CURRENT resting state (`awaiting_answer`/`awaiting_approval`), never an intermediate (`generating_draft`). v1 does NOT rehydrate sessions from disk on service restart ‚Äî documented PRD limitation.
- Session approval slug handling: when the caller passes `slug=None`, the LLM-suggested slug in `draft.meta["slug"]` is piped through `resolve_unique_slug` (silent numeric suffix on collision); when the caller passes an explicit `slug="..."`, it goes through `validate_user_slug` and a collision raises `SlugError` rather than renaming. Matches the PRD's user-vs-LLM asymmetry: users who type a slug expect to see their exact input or an error, not a silent rename.
- `SessionStore` is the single in-memory registry of live sessions. `evict_stale()` is the sweep function ‚Äî it prunes BOTH terminal (completed/cancelled/errored) sessions AND non-terminal sessions older than `ttl_seconds` (default 30 min). Clock is injectable (`clock: Callable[[], float]`) so tests advance time without `time.sleep`. The async cleanup loop itself is deferred to S-013 (`asyncio.create_task` wrapping `evict_stale` on an interval); `cancel_cleanup()` / `cleanup_cancelled()` let the API layer shut down cleanly.
- Session tests never spin up real LLM: `_fake_draft`/`_fake_revise` callables return pre-built `DraftResult` objects. The pattern `session = SynthesisSession(draft_fn=lambda preprocessed, client, *, reader: draft, ...)` plus `session.start_draft()` is the canonical way to drive a session through each state deterministically. `DraftGenerationError(message, attempts=[], last_error=...)` requires keyword args `attempts` and `last_error` ‚Äî raising it with positional args trips the `__init__` signature.
- `synthesizer.revise.generate_revision(*, current_draft, question, answer, client, reader, model=..., max_tokens=..., context_label=...)` is the S-012 phase-2 entry point. Mirrors `generate_draft`'s retry shape (JSON parse / markdown parse / meta validation ‚Üí corrective user turn, capped at `MAX_REVISION_LLM_CALLS=3`), reuses the exact private validators (`_parse_response_json`, `_validate_full_response`, `_ResponseValidationError`) from `synthesizer.draft` so hash-stability of fake-mode retry tests is preserved across phases. Cost & call counts are cumulative: returned `DraftResult.total_cost_usd = current_draft.total_cost_usd + sum(attempt.cost_estimate_usd)`; same additive pattern for `llm_calls`.
- Revision user-message shape: a single `type="text"` block containing four labelled sections ‚Äî `CURRENT_MARKDOWN` (fenced), `CURRENT_META` (pretty-printed JSON), `ANSWERED_QUESTION` (JSON `{id, category, text, answer}`), `REMAINING_QUESTIONS` (JSON array, with the just-answered question filtered out). No screenshots ‚Äî the revision call operates on the current draft text, not the raw trajectory. Tests that compute request hashes should call `synthesizer.revise.build_revision_user_content(current_draft, question, answer)` to build the exact block structure.

- `synthesizer.similarity.score_skill_similarity(generated, golden, client, *, model=SIMILARITY_MODEL, max_tokens=DEFAULT_SIMILARITY_MAX_TOKENS, context_label="synthesizer:similarity") -> SimilarityScore` is the S-014 snapshot-similarity entry point. Uses `SIMILARITY_MODEL = "claude-haiku-4-5"` (5x cheaper than Sonnet; pricing entry already present in `PRICING_USD_PER_MTOK`). Two dimensions (`step_coverage`, `overall`) come from a single Haiku call; two dimensions (`parameter_match`, `destructive_match`) are computed LOCALLY as binary 0.0/1.0 via frozenset equality. Parameter signature uses `(name, type, required)` triple ‚Äî defaults & descriptions intentionally ignored to avoid wording-drift false positives. Destructive signature is the set of destructive step numbers. Strict JSON parsing: no retry on a malformed rubric response ‚Äî raises `SimilarityScoringError`; the design choice (PRD rationale) is that a malformed rubric is a prompt/model issue worth surfacing, not masking.

- Added `services/synthesizer/src/synthesizer/session.py` (~420 lines) exposing `SynthesisSession`, `SessionStore`, `SessionState` (StrEnum), `IllegalStateTransition`, `DraftFn` / `ReviseFn` Protocols, `DEFAULT_SESSION_TTL_SECONDS=1800`, `session_dir()`, `sessions_root()`, plus the `_LEGAL_TRANSITIONS` transition table and `TERMINAL_STATES` frozenset.
- `SynthesisSession` constructor is cheap: allocates a session_id (uuid4 hex), stores reader/client/writer references, sets state=GENERATING_DRAFT. No threading ‚Äî `start_draft()` is explicit so tests can exercise the illegal-transition case (answer_question while state=generating_draft) without racing a background thread.
- State-machine core: `_transition(target)` consults `_LEGAL_TRANSITIONS` and raises `IllegalStateTransition` on illegal moves. Legal flow: GENERATING_DRAFT ‚Üí AWAITING_ANSWER (or AWAITING_APPROVAL if no questions) ‚Üí GENERATING_REVISION ‚Üí AWAITING_ANSWER/AWAITING_APPROVAL ‚Üí SAVING ‚Üí COMPLETED. Any non-terminal state can transition to CANCELLED or ERRORED. Terminal states have no legal outgoing transitions (approve-after-completion, cancel-after-completion both raise).
- Draft phase: `start_draft()` calls `preprocess_trajectory(reader)` then `draft_fn(preprocessed, client, *, reader)` matching `generate_draft`'s signature. Catches `DraftGenerationError` and unexpected exceptions, both route to `_fail(msg)` which sets `error` and transitions to ERRORED. On success, stores the draft, seeds `questions` from `draft.questions`, sets `costs_so_far_usd = draft.total_cost_usd`, transitions to the appropriate next state, then calls `_persist_draft()`.
- Q&A phase: `answer_question(question_id, text)` validates state=AWAITING_ANSWER, non-empty text (raises ValueError on blank), known question_id (raises ValueError on unknown). Transitions to GENERATING_REVISION, calls `revise_fn(current_draft=..., question=..., answer=..., client=..., reader=...)`, catches errors into ERRORED. On success, appends to `answered`, updates draft/questions/costs, then transitions to AWAITING_ANSWER if `next_question() is not None` else AWAITING_APPROVAL. `next_question()` returns the first question whose id isn't in `answered`.
- Approve phase: `approve(slug=None)` validates state=AWAITING_APPROVAL, transitions to SAVING, resolves slug (`resolve_unique_slug` for None, `validate_user_slug` for user-provided), calls `SkillWriter.write(...)`, transitions to COMPLETED and cleans up persistence. `SlugError` from user-provided collision transitions to ERRORED and re-raises so the API layer can return 409. Any other write failure also transitions to ERRORED and re-raises.
- Cancel: `cancel()` transitions any non-terminal state ‚Üí CANCELLED, cleans up `synthesis_sessions/<sid>/` via `shutil.rmtree(..., ignore_errors=True)`. Calling cancel on a terminal state raises `IllegalStateTransition`.
- Persistence: `_persist_draft()` writes `<trace_data_dir>/synthesis_sessions/<session_id>/draft.json` with {session_id, trajectory_id, state, markdown, meta, questions, answered, costs_so_far_usd}. Called AFTER state transitions so the file reflects the resting state, not an intermediate. `snapshot()` returns a UI-facing dict with the same shape plus `next_question` and `error`.
- `SessionStore`: in-memory dict keyed by session_id. `add`/`get`/`remove`/`__contains__`/`__len__`/`ids()` + `evict_stale()` which prunes both terminal sessions and non-terminal sessions older than `ttl_seconds` (default 1800s = 30min). Injectable `clock: Callable[[], float]` for fake-time tests. `cancel_cleanup()`/`cleanup_cancelled()` are hooks for the S-013 async cleanup loop.
- Added `tests/synthesizer/test_session.py` with 23 tests: full happy-path (3 questions ‚Üí approve ‚Üí WrittenSkill), no-questions draft skips straight to AWAITING_APPROVAL, illegal transitions (answer during generating_draft, approve before draft, start_draft after completion, cancel from terminal), cancel from each non-terminal state (generating_draft, awaiting_answer with persistence cleanup, awaiting_approval with no skill written), draft + revision error paths, approve with colliding LLM slug (renames to `_2`), approve with colliding user slug (raises SlugError + ERRORED state), approve with custom user slug (honored verbatim), empty/whitespace answer raises ValueError without changing state, unknown question_id raises, draft persistence file written + updated after revision, SessionStore add/get/remove/duplicate-rejection, evict_stale past TTL with fake clock, evict_stale prunes terminal sessions, cleanup_cancelled flag, snapshot shape progression with state.

  - `Parameter` comparison uses `(name, type, required)` triple ‚Äî NOT including `default` or `description`. Ignoring those fields prevents noise from description-wording drift while still catching behaviorally meaningful parameter changes. If a future use case needs description-aware comparison, add a separate function rather than tightening this one.
  - The test file uses the existing `isolated_data_dir` + `isolated_fakes_dir` pattern from `test_revise.py`. Per-test `TRACE_DATA_DIR` and `TRACE_FAKE_RESPONSES_DIR` env overrides are the canonical way to isolate on-disk writes for LLM tests ‚Äî do NOT roll a new isolation scheme.
  - `pydantic.BaseModel` + `ConfigDict(frozen=True)` + `Field(ge=0.0, le=1.0)` enforces the [0.0, 1.0] constraint at construction time ‚Äî mypy strict is happy with direct float fields, and pydantic raises `ValidationError` on out-of-range values. Frozen models also play nicely with hash/equality-based tests.
  - The Haiku pricing entry already exists in `PRICING_USD_PER_MTOK` since S-006. No pricing-table update needed here ‚Äî that's a signal the table should remain the single source of truth. New models should always land in that dict first, then everything else references it.
