## Codebase Patterns
- Runner uses a **src layout** (`services/runner/src/runner/`). Tests import `from runner.xxx`, resolved via `pythonpath = ["services/runner/src"]` in the root `pyproject.toml` pytest config. Do NOT add a nested `services/runner/runner/` package; that was the old flat layout and has been removed.
- The root pyproject owns the shared `ruff`, `mypy`, and `pytest` config. The runner's own `pyproject.toml` only adds a mypy `files = ["src"]` override so `uv run mypy --strict services/runner` targets the runner package directly.
- Safety gate: every live-mode code path must call `runner.safety.require_live_mode()` in `__init__`. The gate is strict string equality against `TRACE_ALLOW_LIVE="1"` ‚ÄĒ `"true"`, `"yes"`, `"0"`, etc. all disable live mode. `tests/runner/conftest.py` autouses a fixture that force-unsets the flag before every test; opt in with the `live_mode_allowed` fixture.
- Tests mock Anthropic via `respx` (see `anthropic_mock` fixture) ‚ÄĒ never hit the real API.
- Gateway (`gateway/main.py`) imports the runner as `from runner.api import router` (installed-package style via the pytest pythonpath). Synthesizer/recorder are still on the flat layout and import via `from services.{name}.{name}.api` ‚ÄĒ those packages will switch to src layout when their respective branches reach X-001-equivalent stories.
- JSON schemas in `contracts/` are the on-disk contract; the runner pairs each with a pydantic model in `runner.schema` (e.g. `RunMetadata`) for typed in-memory use. Both must agree ‚ÄĒ `extra="forbid"` on the model mirrors `additionalProperties: false` on the schema. Round-tripping uses `model_dump(mode="json", exclude_unset=True)` so unset optional fields do not leak back as nulls.
- Pydantic v2 serializes UTC datetimes with a trailing `Z` (not `+00:00`) via `mode="json"`. Tests that assert round-trip identity must store the canonical `Z` form in fixtures.
- `types-jsonschema` is in the shared dev extras so mypy --strict accepts `jsonschema` imports without module-level overrides.
- Shared contracts that need validation beyond JSON Schema (e.g. list-length, lowercase) live at the repo root under `contracts/` and are policed by `scripts/check_contracts.sh`. Runtime code loads them through a module in `runner.*` with `@lru_cache(maxsize=1)`; the schema path is computed from `Path(__file__).resolve().parents[4]` (for `services/runner/src/runner/*.py`).
- Non-ASCII literals in tests (fullwidth letters, NBSP) should be written as `\uXXXX` escape sequences, not pasted as glyphs ‚ÄĒ ruff's RUF001 flags ambiguous characters and breaks lint otherwise.
- The synthesizer package is now importable at the top level as `synthesizer.*` (not `services.synthesizer.synthesizer.*`). `services/synthesizer` is on both pytest `pythonpath` and mypy `mypy_path`. The recorder is still flat-layout and reached as `services.recorder.recorder.*` until its own X-001-equivalent story lands. Gateway uses `from synthesizer.api import ...` to match.
- Cross-module typechecking: leaving `services/synthesizer/synthesizer` out of mypy `files` while keeping it on `mypy_path` avoids the "source file found twice" ambiguity. Synthesizer code still gets strict-checked because the runner imports it and mypy follows those imports.
- SKILL.md format (locked by X-004 parser): a `# Title` line, a `## Steps` section whose body is `N. text` lines (1-based, contiguous). Destructive steps begin with `‚ö†ÔłŹ ` (fully qualified) or `‚ö† ` (bare). The parser strips the marker from `.text` and sets `destructive=True`. Prose between title and `## Steps` is ignored but preserved on disk.

- Parameter substitution: `meta.parameters[*]` is the declaration. A parameter is required iff it has NO `default` key at all OR `default` is `null`. Unknown parameters from the caller raise `UnknownParameterError`. Escaped braces `\{` / `\}` survive substitution as literal `{` / `}`. Non-string defaults (e.g. integer `30`) are stringified via `str()` before substitution.

Always patch `runner.run_writer.os.fsync` (not `os.fsync`) in tests so the module-local reference is intercepted. `update_status` re-runs pydantic validation via `model_dump() | updates ‚Üí model_validate` ‚ÄĒ `model_copy(update=...)` does NOT re-validate and will happily accept a bogus status enum.

Use `DryRunDisplayInfo` (2880x1800 / 1440x900 / 2.0) as the canonical dry-run display. `round(x)` returns `int` directly ‚ÄĒ never wrap in `int(round(...))` (triggers RUF046).

The `--run-live-input` flag gates `@pytest.mark.live_input` tests via a `pytest_collection_modifyitems` hook that adds a skip marker unless the flag is set.
- For PyObjC-backed modules like `runner.live_input`: import the C constants (e.g. `kCGEventLeftMouseDown`) and functions (`CGEventPost`) at module top level. Tests patch them via `monkeypatch.setattr(live_input_module, "CGEventPost", MagicMock())` ‚ÄĒ the module-local reference is what actually gets called, so patching the Quartz-global won't intercept. `int(kCGEvent*)` coerces PyObjC numeric constants to Python ints when building flag bitmasks with `|`.
- Quartz scroll events: `CGEventCreateScrollWheelEvent(source, units, wheelCount, *wheels)` takes a variadic wheel-value tuple. For vertical scroll use `wheelCount=1, wheels=(delta_y,)`; for horizontal use `wheelCount=2, wheels=(0, delta_x)`. mypy --strict wants the heterogeneous tuple typed as `tuple[int, ...]` before the branching assignment.
- `LiveScreenSource` lives in `runner.live_screen`; Quartz/AppKit symbols (`CGWindowListCreateImage`, `CGPreflightScreenCaptureAccess`, `CGRectInfinite`, `NSBitmapImageRep`, `NSBitmapImageFileTypePNG`) are imported at module top level so tests can `monkeypatch.setattr(live_screen_module, "...", MagicMock())` against the module-local binding. `_cg_image_to_pil` is an instance method rather than a module function so tests can override it via `monkeypatch.setattr(LiveScreenSource, "_cg_image_to_pil", ...)`. Preflight (`CGPreflightScreenCaptureAccess`) is only consulted when capture returns None ‚ÄĒ on the happy path we never pay for it. Untyped PyObjC returns from `CGWindowListCreateImage` need an explicit `image: object | None = ...` annotation before returning, otherwise `mypy --strict` flags a no-any-return warning.
- Agent runtime split: `runner.agent_runtime` defines the `AgentRuntime` Protocol + `AgentResponse` dataclass (pure abstraction, no SDK coupling). `runner.claude_runtime` is the default implementation (`ClaudeRuntime`) plus an `OpenAIRuntime` stub. Fake mode is env-driven (`TRACE_LLM_FAKE_MODE=1`) and matches a `fixtures/llm_responses/runner_<slug>.json` script by `marker` substring in `system_prompt`. Turn selection is by `run_turn` call count (1-based), NOT input hash ‚ÄĒ simpler, deterministic, and tests don't have to maintain hash constants. Cost logging goes to a caller-provided `costs_path`, not a global; tests pass `tmp_path`. The SDK's built-in retry is disabled (`max_retries=0`) so our exponential-backoff policy (1,2,4,8,16s √ó 5 attempts) is the only retry layer; patch `runner.claude_runtime.asyncio.sleep` in tests to skip the real waits. The `computer_20250124` tool's `display_width_px`/`display_height_px` come from `current_image_mapping.resized_pixels` when set, else from `DryRunDisplayInfo` downscaled to longest-edge 1568 (= 1568√ó980) ‚ÄĒ call `runtime.set_image_mapping(...)` after each screenshot.
- Anthropic SDK exception taxonomy: `AuthenticationError` subclasses `APIStatusError`, so `except AuthenticationError` MUST come before `except APIStatusError` in the `try` block ‚ÄĒ the reverse order would swallow 401s into the retry branch. `RateLimitError` (429) also subclasses `APIStatusError`; catch via `APIStatusError` and branch on `.status_code`.
- Anthropic `Message.content` is a list of typed block objects (TextBlock, ToolUseBlock, ‚Ä¶). Convert to plain dicts via `[block.model_dump() for block in msg.content]` before handing off to downstream parsers ‚ÄĒ keeps `AgentResponse.content_blocks` JSON-serializable and matches the fake-mode fixture shape one-for-one.
- Execution prompt (`runner.execution_prompt`) is load-bearing for safety and must stay deterministic. Protocol strings (`DESTRUCTIVE_PROTOCOL`, `COMPLETION_PROTOCOL`, `ERROR_PROTOCOL`, `VERIFICATION_PROTOCOL`, `DRY_RUN_NOTICE`) are module-level `Final` constants so downstream parsers (X-012) and tests pin them by reference ‚ÄĒ never copy-paste the strings. The literal tag `<needs_confirmation step="N"/>` is the single source of truth; the harness-layer keyword matcher (X-015) is the belt-and-suspenders backup.
- Rendering a `ParsedSkill` back to markdown: the parser strips the `‚ö†ÔłŹ ` prefix and exposes `step.destructive` separately. When re-rendering for the prompt, re-attach an explicit `‚ö†ÔłŹ [DESTRUCTIVE] ` prefix so the rendered text matches the `DESTRUCTIVE_PROTOCOL` instruction wording verbatim.
- Dispatcher (`runner.dispatcher`) translates a `ToolCallAction` into `InputAdapter` + `ScreenSource` calls and returns a `ToolResult(content_blocks, new_image_mapping, is_error)`. Every side-effecting action (click/type/key/scroll/wait) ends with a fresh screenshot; `mouse_move` is the sole exception. `wait` duration is capped at `MAX_WAIT_SECONDS=10.0`. Coordinates arrive in resized-pixel space and are mapped to display points via `resized_pixels_to_points(..., image_mapping)` inside the dispatcher ‚ÄĒ callers pass the `image_mapping` from the most recent screenshot. `double_click` is implemented as two `click()` calls on the same point (InputAdapter has no click-count param). Click `text` field (e.g. `"cmd+shift"`) is parsed via `parse_key_expression` into a modifier list. The dispatcher never raises on malformed tool_input ‚ÄĒ bad `action`, missing/negative/out-of-bounds coordinates, non-string `text`, wrong `scroll_direction`, negative `scroll_amount`/`duration`, unknown action name all return `ToolResult(is_error=True, content_blocks=[{type:"text", text:"..."}])` WITHOUT calling the adapter. Booleans are explicitly rejected in numeric fields (coordinate, scroll_amount, duration) even though `bool` subclasses `int`.
- Tool-call parser (`runner.parser`) is the first destructive gate at the text-tag level: `parse_agent_response(AgentResponse) -> ParsedAction` returns a tagged union of `ToolCallAction | ConfirmationRequest | WorkflowComplete | WorkflowFailed | UnknownAction`. Priority order in same turn: `ConfirmationRequest` > `WorkflowComplete` > `WorkflowFailed` > `ToolCallAction` > `UnknownAction`. A confirmation tag ALWAYS drops any concurrent `tool_use` block (safety). A malformed confirmation (missing quotes, extra attrs, step <= 0, non-integer) also drops the tool_use ‚ÄĒ we detect the attempt via a loose `<\s*needs_confirmation\b` pattern and return `UnknownAction` rather than silently falling through. Parser never raises. Content blocks are duck-typed as dicts (X-010 converts via `model_dump()`); real SDK block objects are not supported.
- Run budgets (`runner.budget`): `RunBudget` (frozen dataclass, five caps, defaults 500k/50k in-out tokens, 600s wall clock, 30 actions/min, 100 total) + `BudgetTracker` (mutable). Hard limits (tokens/time/total) return `BudgetStatus.budget_exceeded(reason)` ‚Üí executor aborts; soft per-minute cap returns `rate_limited(wait_seconds)` ‚Üí executor sleeps. `time_source` is injectable (defaults to `time.monotonic`); tests pass a `FakeClock` callable instead of monkeypatching. Sliding window is a `deque[float]` trimmed in `check()` with `oldest <= cutoff` (60s old ‚Üí expired). Per-skill overrides load via `RunBudget.from_skill_meta(meta)` reading optional `meta["runtime_limits"]`; unknown nested keys are rejected both python-side AND by `skill-meta.schema.json`'s `additionalProperties: false` on the nested object. `"runtime_limits": null` must no-op to defaults, not error. Bool-rejection (`isinstance(x, bool)` before the numeric check) guards every int-typed input.
- Executor (`runner.executor`) is the X-017 orchestration boundary: per turn ‚Üí budget check ‚Üí `agent_runtime.run_turn` ‚Üí accumulate tokens/cost ‚Üí parse ‚Üí dispatch per action kind. Finalisers (`_finalize_succeeded|failed|aborted|budget_exceeded|agent_stuck`) are the only paths that set terminal status; every exit goes through `RunWriter.update_status` so on-disk metadata stays schema-valid. Image mapping starts at a `_default_image_mapping()` seeded from `DryRunDisplayInfo` downscaled to 1568 longest edge (matches `ClaudeRuntime._default_display_dims()` exactly) and updates every time the dispatcher returns an `image` content block. Fourth consecutive `UnknownAction` trips `agent_stuck` with `error_message=AGENT_STUCK_REASON` so a model narrating without acting cannot burn tokens forever. Dry-run and execute share the same executor ‚ÄĒ only the prompt mode and the gate's dry-run short-circuit differ.
- ConfirmationQueue (`runner.confirmation`): backed by `dict[run_id, asyncio.Queue[ConfirmationDecision]]` with `maxsize=1`. `push_request(*, run_id, step_number, step_text, screenshot_ref, destructive_reason) -> ConfirmationRequest` raises `IllegalState` on double-push; `await_decision(run_id, timeout_seconds=300.0)` synthesizes `ConfirmationDecision(action='abort', reason='user_timeout')` on timeout and cleans up the pending + channel dicts in `finally`; `submit_decision(run_id, decision) -> bool` warning-logs and returns False when there is no pending request (legitimate race with timeouts / kill_run); `kill_run(run_id, reason='kill_switch')` wraps `submit_decision` with a synthetic abort for X-018. The queue's `ConfirmationRequest` dataclass shares its name with `runner.parser.ConfirmationRequest` on purpose ‚ÄĒ the parser one is the agent's tag-level signal, the queue one carries the full UI prompt (step_number+step_text+screenshot_ref+destructive_reason); callers needing both alias on import. WS shapes: `make_request_message(request, screenshot_url=...)` ‚Üí `{type:'confirmation_request', run_id, step_number, step_text, destructive_reason, screenshot_url}`; `parse_response_message(payload) -> (run_id, ConfirmationDecision)` validates every field and raises `ValueError` on drift. Use builtin `TimeoutError` not `asyncio.TimeoutError` (ruff UP041).

- **Learnings for future iterations:**
  - The default `ImageMapping` and `ClaudeRuntime._default_display_dims` must stay in lockstep ‚ÄĒ both derive from `DryRunDisplayInfo` downscaled to longest-edge 1568. If one drifts, the very first tool call decodes Claude coords against different dims than the model was shown, producing mis-clicks. Extract the constant if another module needs it.
  - `TrajectoryScreenSource` loads the 1440√ó900 blank canvas fallback which is already at/under the 1568 cap, so `capture_and_normalize` does NOT downscale it and the resulting `scale_from_resized_to_points = (1440/1440) / 2.0 = 0.5`. Tests that assert on display-point coords after a trajectory screenshot must use 0.5√ó the resized coord, NOT the default-mapping 0.918√ó. This is a common test trap ‚ÄĒ the mapping changes the moment the first screenshot is dispatched.
  - `asyncio.TaskGroup` (3.11+) is the clean way to drive `executor.run()` alongside a sidecar confirm/abort task: start both tasks, the group waits for both, and a failure in either propagates. Polling pattern: `while not queue.has_pending(run_id): await asyncio.sleep(0)` lets the executor reach `await_decision` before the test submits.
  - `RunWriter.update_status` re-runs pydantic validation via `model_dump() | updates ‚Üí model_validate`. The executor's `_snapshot_metadata` helper always seeds `status="running"`; the real status comes from the writer call's `status=` kwarg. This double-setting is intentional: it keeps the seed valid while letting the final status land via the validating update path.
  - `datetime.now(UTC)` (ruff UP017) ‚ÄĒ `timezone.utc` is discouraged in py3.11+.
  - When a class uses forward references (`_GateAbort`/`_GateAllow` defined below `Executor`), the `from __future__ import annotations` import makes quotes unnecessary (ruff UP037). Just name the class ‚ÄĒ the annotation is a string anyway.
