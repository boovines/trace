## Codebase Patterns
- `services/recorder` uses a **src layout** (`services/recorder/src/recorder/ŌĆ”`); future files (schema.py, writer.py, session.py, etc.) go under `src/recorder/`.
- The recorder package is installed as a uv workspace member (`trace-recorder`) and should be imported as `from recorder.xyz import ŌĆ”`, NOT `services.recorder.recorder.xyz`.
- Root `pyproject.toml` owns shared tooling config (ruff/mypy/pytest); per-service `pyproject.toml` only declares that service's runtime deps + hatchling build.
- Ruff's isort needs `known-first-party = ["recorder", "synthesizer", "runner", "gateway"]` or it misgroups the workspace imports in gateway/main.py.
- PyObjC wheels only exist for macOS ŌĆö deps are marked `; sys_platform == 'darwin'` so `uv sync` still works if Ralph ever runs on non-darwin.
- `mypy --strict` has PyObjC module overrides in root pyproject (`Cocoa.*`, `Quartz.*`, `ApplicationServices.*`, `AppKit.*`, `Foundation.*`, `objc.*`); add new PyObjC module imports here, not in each file.
- Quality-gate commands for the recorder module are:
  - `uv run ruff check services/recorder`
  - `uv run mypy --strict services/recorder`
  - `uv run pytest tests/recorder/`
- Pre-existing gateway/integration test failures (`ModuleNotFoundError: gateway`) are NOT caused by R-001 and are outside this branch's scope ŌĆö a future `feat/integration` story will install gateway as a proper package.
- **Schema validation**: `recorder.schema` exposes `validate_event(event)` / `validate_metadata(meta)` backed by `jsonschema`'s Draft 2020-12 validator (with `FORMAT_CHECKER` enabled for `uuid`/`date-time`). Future writer/session code should call these before writing JSON to disk so bad events never land in `events.jsonl`.
- **Schema file lookup**: `recorder.schema.find_schema_path()` walks up from `__file__` to find `contracts/trajectory.schema.json`. Works for the uv-workspace editable install. If the contracts ever move or get bundled into the wheel, update this function ŌĆö do NOT hardcode a path.
- **Local `$ref` only**: the `load_event_schema()` / `load_metadata_schema()` wrappers build standalone schemas with a `#/$defs/...` fragment ref (no `$id`), which keeps `jsonschema` from trying to HTTP-fetch the schema's `$id` URL at validate time. If you add new top-level sub-schemas for Synthesizer/Runner to reuse, follow the same pattern.
- **TrajectoryWriter ownership**: `recorder.writer.TrajectoryWriter` is the only module allowed to mutate the on-disk trajectory directory. All writes go through its lock; session/orchestrator code must not write `events.jsonl` / `metadata.json` / `screenshots/*` directly. It validates via `recorder.schema.validate_{event,metadata}` BEFORE any disk write, so bad data never leaks to disk. The constructor creates `<root>/<uuid>/` + `screenshots/` with perms `0700` and the writer is usable as a context manager (`__exit__` calls `close()`).
- **Atomic JSON writes**: `_atomic_write_json(path, data)` writes to `<path>.tmp`, fsyncs, and renames. Use it for any metadata-style file. `events.jsonl` is append-flush (no fsync per line ŌĆö documented v1 tradeoff).
- **Datetime import style**: use `from datetime import UTC, datetime` (Python 3.11+). Ruff's `UP017` flags `timezone.utc` ŌĆö don't revive it. ISO strings are produced with `datetime.now(UTC).isoformat(timespec="milliseconds")` which yields `+00:00` suffixes; the schema's `format: date-time` accepts both `+00:00` and `Z`.
- **Ruff SIM105**: prefer `with contextlib.suppress(OSError): os.fsync(...)` over `try/except/pass` for best-effort fsync on filesystems that don't support it (tmpfs in pytest tmp_path).
- **PyObjC mocking in tests**: for any module that does `from ApplicationServices import X` / `from Quartz import Y` inside a function, tests stub the framework by installing a `types.ModuleType("Quartz")` / `types.ModuleType("ApplicationServices")` into `sys.modules` via `monkeypatch.setitem`. The import inside the tested function then resolves to the stub. For ImportError coverage, install a stub that is missing the expected attributes ŌĆö `from X import missing_attr` raises ImportError, which the module under test should catch.
- **Permission-check pattern**: `recorder.permissions` keeps each macOS framework call in its own thin `_check_*` helper, and `check_permissions()` / `get_missing_permissions_error()` simply compose them. Tests patch the helpers for combination coverage and patch the PyObjC symbols inside `sys.modules` for helper-level coverage. Use the same pattern for any future PyObjC-backed module (event_tap, ax_resolver, focus_tracker, screenshot) to keep the suite hermetic on Ralph's sandbox.
- **`@pytest.mark.macos` is opt-in only**: `tests/recorder/conftest.py` auto-skips every test tagged `macos` unless `TRACE_RUN_MACOS_TESTS=1` is set. Use this marker for ANY test that needs real PyObjC framework behaviour (CGEventTap, AXUIElement, CGWindowListCreateImage, etc.). Hermetic unit coverage should still be provided alongside via `sys.modules`-stubbed Quartz/ApplicationServices so the default run stays green.
- **Splitting PyObjC callback from normaliser**: for modules driven by a C-callback (CGEventTap), put the pure eventŌåÆdict conversion in a module-level function that takes the framework module as a parameter (`_normalise_event(code, event, quartz)`), and expose a thin `handle_cg_event(code, event)` method on the class that the tap callback delegates to. Tests exercise the normaliser + re-enable logic by calling `handle_cg_event` with fake events ŌĆö no run-loop or thread needed. `event_tap.py` is the reference implementation.
- **Re-enabling a disabled tap**: `CGEventTapEnable(tap, True)` re-arms a tap the OS disabled via `kCGEventTapDisabledByTimeout` / `kCGEventTapDisabledByUserInput`. Both constants appear as unsigned `uint32` (0xFFFFFFFE / 0xFFFFFFFF) in headers but PyObjC sometimes surfaces them as signed ints (-2 / -1); match both forms.
- **Bounded-time framework calls**: when a PyObjC call could block indefinitely (AX queries, inter-app RPC), wrap it with a daemon `threading.Thread` and a `threading.Event` set in a `finally`. The caller `event.wait(timeout)` ŌĆö on timeout the worker thread just keeps living as a daemon. Do not rely on `Thread.join(timeout)` alone to tell "done" from "still running"; always use a separate event sentinel. See `ax_resolver.resolve_element_at` for the reference.
- **PyObjC AX value shapes**: AX position/size attributes come back as one of three shapes depending on macOS/PyObjC version ŌĆö native 2-tuple, bridged object with `.x/.y` or `.width/.height`, or raw `AXValueRef` that must be unwrapped via `AXValueGetValue(value, kAXValueCG{Point,Size}Type, None) -> (success, unpacked)`. Any new AX-consuming code should tolerate all three (see `_unpack_ax_value` in ax_resolver.py).
- **NSWorkspace subscription pattern**: prefer `addObserverForName:object:queue:usingBlock:` (a Python callable passed as block) over the NSObject-subclass selector pattern ŌĆö it keeps the handler method on the Python class and avoids `@objc.selector` machinery. The block returns an opaque observer handle that must be passed back to `removeObserver:` on stop. Tests stub `AppKit.NSWorkspace.sharedWorkspace().notificationCenter()` with a class that captures `(name, block)` in a list so we can invoke the block synchronously with a fake notification. See `focus_tracker._subscribe_workspace` for the reference.
- **NSDictionary dual-access**: `userInfo()` on an `NSNotification` is bridged to a Python dict in most PyObjC builds but not all. Accept both by trying `.get(key)` first (callable on a bridged dict) and falling back to `objectForKey_(key)` on an ObjC-only stub. Same pattern applies to any NSDictionary the recorder reads out of a framework callback.
- **Idle-flush timer pattern**: for any module that aggregates into a buffer until a "nothing happened for N seconds" signal, use a `threading.RLock` around the state + a `threading.Timer` that is cancelled-and-replaced on every ingest event. The RLock (not a plain Lock) is required because the Timer's callback may take the lock while a caller-driven `flush()` is mid-flight. The Timer callback should also check an `is_stopped` flag so a late-firing cancelled timer is a silent no-op. `text_aggregator.py` is the reference.
- **Session-layer translation of event_tap output**: `recorder.event_tap` emits modifier names in PyObjC framework style (`"command"`, `"control"`, `"option"`, ŌĆ”). The trajectory schema's `modifier` enum uses short forms (`"cmd"`, `"ctrl"`, `"opt"`, ŌĆ”). Translation happens in the session orchestrator (R-010) when shaping a schema-valid `keypress` event. Downstream consumers of event_tap (text_aggregator, session) must agree on which name style they expect ŌĆö currently everything inside the recorder uses the long PyObjC names; ONLY the session's final schema-event builder maps to the short enum.
- **Stateless policy modules**: `recorder.keyframe_policy.KeyframePolicy` is stateless ŌĆö the session passes `(event_type, seconds_since_last_keyframe)` each call and the policy answers yes/no + reason. The session owns the "time since last keyframe" state and the periodic timer. Prefer this shape for any future rule-based gate (e.g. scroll-debounce) ŌĆö stateless is easier to test, thread-safe by construction, and doesn't couple the rule to a particular timer impl.
- **Class-level state in PyObjC test doubles**: when emulating the `cls.alloc().initŌĆ”_(ŌĆ”)` two-step idiom, do NOT use mutable class attributes at the base class (RUF012). Either put state on the instance, or declare `ClassVar[...]` explicitly, or create a fresh subclass per test via `type("Name", (Base,), {...})` ŌĆö the screenshot stub uses the subclass trick so each test's `rep_calls` list is isolated.
- **PNG validation without Pillow**: the `recorder.screenshot` module exposes `PNG_MAGIC = b"\\x89PNG\\r\\n\\x1a\\n"` and rejects bytes that don't start with it. Tests synthesize a minimal valid PNG by hand (magic + IHDR + one compressed IDAT + IEND, with `zlib.crc32` for the per-chunk CRC) ŌĆö avoid adding Pillow just to verify PNG shape. Dimension parsing uses `struct.unpack(">II", png[16:24])` once you've asserted `png[12:16] == b"IHDR"`.
- **RecordingSession owns `seq`**: `recorder.session.RecordingSession` is the single place where `seq` values are minted for a running trajectory. `TrajectoryWriter` still has its own auto-seq counter (belt-and-suspenders), but every normal emit path passes a pre-assigned seq under the session's `_lock`. Any future capture path that does NOT go through the session (e.g. an HTTP admin endpoint that injects events) must either acquire the same lock or accept interleaved seqs.
- **Session dependency injection**: every macOS-dependent piece of `RecordingSession` is injected via a constructor kwarg (`event_tap_factory`, `focus_tracker_factory`, `text_aggregator_factory`, `permissions_check`, `ax_resolver`, `screenshot_capturer`, `display_info_provider`, `keyframe_policy`). Hermetic tests pass fakes; real use passes nothing and defaults fall back to the real modules. Keep this pattern for any future orchestrator ŌĆö it is the only way tests can run in Ralph's sandbox without PyObjC.
- **Dual-lock orchestrator pattern**: `_start_stop_lock` serialises `start()`/`stop()` bookkeeping; `_lock` guards per-event state (seq, writer.append_event, last-keyframe-time, pending timers). Do NOT collapse them ŌĆö `stop()` spends non-trivial time joining threads and holding that on the per-event lock would starve capture callbacks. Capture-thread callbacks check `self._active` under `_lock` and bail if False, so late callbacks after `stop()` are silent no-ops.
- **FastAPI `Depends` pattern**: bind `Depends(...)` once at module level (`StateDep = Depends(get_recorder_state)`) and reuse it as each endpoint's default ŌĆö ruff `B008` rejects inline `Depends(...)`, but FastAPI inspects the default for the `Depends` instance regardless. This is the canonical workaround.
- **Hermetic FastAPI testing**: drive the app via `httpx.AsyncClient(transport=ASGITransport(app=app))` (no real socket) and override dependencies via `app.dependency_overrides[get_recorder_state] = lambda: state`. Do NOT mutate module-level singletons in tests ŌĆö they leak across tests because pytest-asyncio reuses the event loop.
- **HTTP path traversal testing**: `httpx.AsyncClient` normalises `..`/`.`/empty path segments BEFORE the request reaches the ASGI app, so HTTP-level traversal tests are unreliable. Unit-test the path-safety helper (e.g. `_safe_screenshot_path`) directly with raw filename strings instead of through the HTTP layer.
- **Storage root resolution**: `recorder.storage.default_trajectories_root()` resolves prod (`Trace`) vs dev (`Trace-dev`) profile via `TRACE_DEV_MODE=1`. Tests pin a `tmp_path` via `TRACE_PROFILE_DIR`. Any new module that needs to write to the per-user app data directory should reuse this helper rather than computing its own paths.
- **PermissionsError forwarding**: `PermissionsMissingError.error` is the structured `PermissionsError` TypedDict ŌĆö return it verbatim as the 403 body via `JSONResponse(status_code=403, content=dict(exc.error))`. Synthesizer/Runner endpoints that need similar structured-error responses should follow the same shape rather than re-inventing.
- **IndexDB is a cache, not a source of truth**: `recorder.index_db.IndexDB` mirrors per-trajectory summary fields in SQLite for fast `GET /trajectories`. The filesystem (`metadata.json` + `events.jsonl`) remains authoritative; `IndexDB.reconcile(root)` rebuilds the cache from disk and is invoked automatically in `RecorderState.__init__` so a service restart heals drift without manual intervention. Writer failures against the index are silently suppressed via `contextlib.suppress(Exception)` ŌĆö a cache outage must not break a live recording.
- **SessionFactory signature includes IndexDB**: `recorder.api.SessionFactory = Callable[[Path, IndexDB | None], RecordingSession]`. Any test that substitutes `session_factory=FakeSession` must have `FakeSession.__init__(self, root, index_db=None)`, and should forward the index_db to its `TrajectoryWriter` so `list_trajectories` assertions see the rows. Changing the factory again later is a breaking change across every test suite that mocks a session ŌĆö think twice.
- **Where the index lives**: `<app_support>/index.db` (i.e., `default_trajectories_root().parent / "index.db"`), NOT inside the `trajectories/` directory.  `recorder.storage.default_index_db_path()` is the canonical resolver ŌĆö reuse rather than hardcoding. Tests pin via the same `TRACE_PROFILE_DIR` override that controls `default_trajectories_root`.
- **duration_ms tests must tolerate sign**: `TrajectoryWriter.close()` rewrites `stopped_at` to `now()`, so a hermetic test whose `started_at` is a fixed future date will produce a negative `duration_ms`. Either assert on presence-not-sign (`is not None`), OR overwrite the index row post-close with deterministic values (that's what `FakeSession.stop()` in `test_api.py` does ŌĆö then `list_trajectories` can assert `duration_ms == 1500`).
- **Fixtures are synthetic until a human re-records**: `fixtures/trajectories/<slug>/` is produced by `scripts/generate_synthetic_fixtures.py` (Ralph/CI path). Real recordings replace them during smoke testing per `scripts/regenerate_fixtures.sh`. Downstream tests that depend on fixtures must rely only on structural contracts (schema validity, per-type event counts) ŌĆö never on the literal text of a `text_input` or the pixel contents of a screenshot. `tests/recorder/test_fixtures.py` is the reference structural check.

- **Learnings for future iterations:**
  - `datetime.now(timezone.utc)` trips ruff `UP017` on Python 3.11+ ŌĆö use `from datetime import UTC` and `datetime.now(UTC)`.

## Codebase Patterns
- Runner uses a **src layout** (`services/runner/src/runner/`). Tests import `from runner.xxx`, resolved via `pythonpath = ["services/runner/src"]` in the root `pyproject.toml` pytest config. Do NOT add a nested `services/runner/runner/` package; that was the old flat layout and has been removed.
- The root pyproject owns the shared `ruff`, `mypy`, and `pytest` config. The runner's own `pyproject.toml` only adds a mypy `files = ["src"]` override so `uv run mypy --strict services/runner` targets the runner package directly.
- Safety gate: every live-mode code path must call `runner.safety.require_live_mode()` in `__init__`. The gate is strict string equality against `TRACE_ALLOW_LIVE="1"` ŌĆö `"true"`, `"yes"`, `"0"`, etc. all disable live mode. `tests/runner/conftest.py` autouses a fixture that force-unsets the flag before every test; opt in with the `live_mode_allowed` fixture.
- Tests mock Anthropic via `respx` (see `anthropic_mock` fixture) ŌĆö never hit the real API.
- Gateway (`gateway/main.py`) imports the runner as `from runner.api import router` (installed-package style via the pytest pythonpath). Synthesizer/recorder are still on the flat layout and import via `from services.{name}.{name}.api` ŌĆö those packages will switch to src layout when their respective branches reach X-001-equivalent stories.
- JSON schemas in `contracts/` are the on-disk contract; the runner pairs each with a pydantic model in `runner.schema` (e.g. `RunMetadata`) for typed in-memory use. Both must agree ŌĆö `extra="forbid"` on the model mirrors `additionalProperties: false` on the schema. Round-tripping uses `model_dump(mode="json", exclude_unset=True)` so unset optional fields do not leak back as nulls.
