## Codebase Patterns
- uv workspace: dev extras on workspace members aren't picked up by `uv sync --extra dev` at the root; use `uv sync --all-extras --all-packages` (or run per-package) to install a workspace member's dev deps.
- Synthesizer package uses **src layout** (`services/synthesizer/src/synthesizer/`) with hatchling; declare `[tool.hatch.build.targets.wheel] packages = ["src/synthesizer"]` so the wheel finds it.
- `pytest` exits 5 on "no tests collected" — that's NOT zero. If a story's AC requires `pytest ... exits 0`, add at least one smoke test to the new suite.
- All synthesizer tests run with `TRACE_LLM_FAKE_MODE=1` + `ANTHROPIC_API_KEY=test-dummy-key` forced via an autouse fixture in `tests/synthesizer/conftest.py`, plus the `anthropic_mock` respx router is autouse-installed on every test so an unmocked call to `api.anthropic.com` raises instead of hitting the network.
- Per-service `pyproject.toml` declares its own `[tool.mypy] strict = true` so `uv run mypy --strict services/synthesizer` uses service-scoped config (not the root's). Root `[tool.mypy]` still covers `services` + `gateway` for repo-wide checks.
- `contracts/*.schema.json` is located at runtime by walking up from `Path(__file__)` until a directory containing `contracts/` is found; this keeps modules working from both the editable workspace install and a future published wheel without hard-coding a relative path.
- jsonschema's `FormatChecker` validates `uuid` out-of-the-box, but `date-time` requires the optional `rfc3339-validator` dep — schemas relying on strict date-time enforcement should combine `type: "string"` with a regex pattern or a post-validation Python check, not just the `format` keyword.
- `jsonschema` ships no PEP 561 type stubs; use `# type: ignore[import-untyped]` on the import line to keep `mypy --strict` green without pulling in `types-jsonschema`.
- When two branches of one function both assign `sorted(set_diff)` to the same variable name, mypy pins the variable's element type from the first assignment. Use distinct names per branch (e.g., `only_md_steps` vs `only_md_params`) to avoid spurious `incompatible type` errors.
- SKILL.md format (locked by S-003): sections in strict order `# Title` → description → `## Parameters` → `## Preconditions` → `## Steps` → `## Expected outcome` → optional `## Notes`. Parameter bullet shape: `` - `name` (type, required|optional[, default: value]) — description ``. Defaults: strings double-quoted, integers bare, booleans `true`/`false`. Empty bullet sections render `_None._`. Destructive steps prefixed with `⚠️ [DESTRUCTIVE]` exactly.
- Pydantic v2 `model_config = ConfigDict(frozen=True)` on `ParsedSkill`/`Step`/`Parameter` makes round-trip tests use bare `==` for equality and also prevents accidental mutation by downstream stages.
- The parameter-reference regex `(?<!\\)(?<!\{)\{NAME\}(?!\})` plus pre-stripping fenced/inline code is the approved pattern for finding `{name}` refs — mirrors S-002 schema cross-check but is stricter (handles escaped `\{`, `{{double}}`, inline backticks). Use `extract_parameter_refs` from `skill_doc` rather than re-rolling the regex in future stories.
- The trajectory schema (`contracts/trajectory.schema.json`) requires every event to carry `seq` (integer ≥ 1, unique per trajectory), and supports `click` and `app_switch` event kinds plus an optional `target: {label, role, bundle_id}` sub-object on clicks. Downstream stories (S-005 preprocessor, S-008 destructive matcher) read click `target.label` — populate it on every click event in fixtures.
- Reference-fixture trajectories live at `fixtures/trajectories/<slug>/` with `metadata.json`, `events.jsonl`, and `screenshots/0001.png` (a 67-byte 1×1 grayscale PNG). Tests that mutate a fixture MUST `shutil.copytree` it into `tmp_path` first — the committed fixtures are ground truth for S-016 and other downstream tests.
- `TrajectoryReader` validates `metadata.json` against `definitions.metadata` and each `events.jsonl` line against `definitions.event` using two separate `Draft202012Validator` instances, cached via `lru_cache`. Error messages always include the line number AND the event's `seq` so a malformed fixture is trivial to locate.
- Event objects use `ConfigDict(extra='allow', frozen=True)` — same pattern as `ParsedSkill`. Unknown keys flow through for forward-compat, and frozen events are safe to cache / hash / compare in downstream stages.
- Events are sorted by `seq` on load, so `iter_events()` / `iter_events_by_type()` always yield in logical order regardless of on-disk line order. Duplicate `seq` values raise on construction (would make `get_screenshot_path` ambiguous).
- Trajectory digest (`synthesizer.preprocess.DigestEntry`) is the single shape fed to the draft-generation prompt. `seq=0` on a digest entry marks a synthetic idle gap (never a real event since the schema's seq minimum is 1); `scroll_run` entries reuse the first collapsed scroll's seq. Downstream stories (S-007 draft prompt, S-012 revision) should treat `seq=0`/`kind in {scroll_run, idle}` as synthetic and not try to trace them back to a single event.
- `synthesizer.preprocess.APP_NAMES` maps common macOS bundle_ids to human names (Chrome, Finder, Notes, Slack, ...). Unknown bundle_ids fall through to the bundle_id itself — extend the map here rather than adding per-call branches in prompt code.
- The preprocessor drops `mouse_move`/`mouse_down`/`mouse_up` as noise but keeps every `click`, `text_input`, and `app_switch` exactly once. When constructing synthetic trajectories in tests, omit mouse_move/down/up entirely unless you specifically want to exercise the noise filter.
- The synthesizer never imports the Anthropic SDK directly — all calls go through `synthesizer.llm_client.LLMClient`. That wrapper owns retries (the SDK's `max_retries=0`), fake mode, and cost logging in one place. Downstream stories (S-007 draft, S-012 revision, S-014 similarity) MUST construct one `LLMClient` per session and pass it down rather than instantiating per call — keeps the cost log coherent and makes the no-stray-network assertion meaningful.
- Trace data dir resolution is `synthesizer.llm_client.trace_data_dir()`: honors `TRACE_DATA_DIR` env override (used by tests with `tmp_path`), then falls back to `TRACE_PROFILE` selecting `Trace` (prod) vs `Trace-dev` (dev), defaulting to prod. New code that writes anywhere under `~/Library/Application Support/Trace[-dev]/` should call this helper rather than re-deriving the path.
- Pricing for cost estimates lives in `PRICING_USD_PER_MTOK` in `llm_client.py` — single source of truth. Add new model entries here when bumping; the test `test_pricing_table_has_required_models` enforces presence of Sonnet 4.5 + Haiku 4.5 entries.
- Fake-mode canned responses are keyed by `compute_request_hash(system, messages, model, max_tokens, tools)` (sha256 over canonical JSON). Tests register them with `save_fake_response(...)` against a per-test `TRACE_FAKE_RESPONSES_DIR` so they don't leak across tests; downstream story integration tests that need committed fixtures live under `fixtures/llm_responses/`.
- `costs.jsonl` line shape is locked: `{timestamp_iso, module, model, input_tokens, output_tokens, cost_estimate_usd, context_label}`. S-018 (budget guardrails) reads this back, so don't drop or rename keys.
- `LLMResponse.cost_estimate_usd` is the per-call figure *as billed* — always `0.0` in fake mode, matched 1:1 to the `costs.jsonl` line for that call. Downstream code that accumulates cost across a session (S-007 DraftResult, S-012 revision, S-018 budget check) should sum this field rather than re-estimating from `input_tokens` + model, because a re-estimate would double-bill fake-mode tests at Sonnet prices.
- Draft-generation user message shape: a single `user` content block list containing exactly one `type="text"` digest block followed by up to 20 `type="image"` base64 PNG blocks. `synthesizer.draft.build_user_content(preprocessed, reader)` is the single builder — call it from tests that need to compute request hashes, never hand-roll the blocks.
- Draft retry chain: on any validation failure (`_ResponseValidationError`), `generate_draft` appends the bad response as an `assistant` turn and the feedback string as the next `user` turn, then re-calls. Hash stability for canned responses relies on the feedback strings being pure functions of the bad response — if S-012 reuses this retry shape, keep `_parse_response_json`/`_validate_markdown`/`_validate_meta_and_cross` as the single source of feedback text.
- `fixtures/prompt_examples/*.txt` holds the few-shot examples embedded in `DRAFT_SYSTEM_PROMPT` at import time; files are read in sorted filename order for determinism. Changing the examples silently invalidates every canned fake-mode response hash — regenerate them in lockstep or commit both together.
- `[tool.ruff.lint.per-file-ignores]` at the repo root now has an E501 exemption for `draft_prompt.py` — long prose inside the embedded LLM system prompt. When adding new LLM-prompt modules, add them to the same per-file-ignores rather than hand-wrapping paragraphs (hard wraps degrade prompt readability for the model).
- `synthesizer.destructive_matcher.apply_destructive_matcher` uses **label-in-step-text binding**, not positional index-to-click pairing: for each click with a destructive label, find the earliest un-bound step whose text case-insensitively contains the label substring. Positional pairing is wrong because step lists include non-click entries (app_switch, text_input) that misalign the index with `iter_events_by_type("click")`. The digest's click step summaries (``Clicked button labeled "Send" in Chrome``) embed the label, so substring match finds the right step reliably. Downstream: if the draft prompt ever stops including the literal label in step text, the matcher silently stops flagging — keep the label in the summary.
- `synthesizer.slug`: canonical slug shape is `^[a-z][a-z0-9_]{2,39}$` (constants `SLUG_PATTERN`, `SLUG_REGEX`, `SLUG_MIN_LEN=3`, `SLUG_MAX_LEN=40`). `slugify(name)` pipeline: NFKD transliterate → lowercase → collapse any `[^a-z0-9_]+` run into single `_` → strip leading non-letters (digits, underscores, leftover junk) → rstrip `_` → truncate to 40, rstrip `_` again → error if <3 chars. `resolve_unique_slug(base, dir)` only uses numeric suffixes (`base_2`, `base_3`, ...) — the `_v2`/`_copy`/`_new` naming ambiguity is disallowed by design. `validate_user_slug(input, dir) -> (bool, reason)` returns human-readable rejection messages suitable for surfacing to the user in the Q&A UI. Downstream (S-010 writer, S-011 session approve, S-013 API `/approve` endpoint) should always call `resolve_unique_slug` before a disk write and prefer `validate_user_slug` before accepting a user-provided override.

## 2026-04-23 - S-009 Slug generation and collision handling
- Added `services/synthesizer/src/synthesizer/slug.py` (~190 lines) exposing `slugify`, `resolve_unique_slug`, `validate_user_slug`, `SlugError`, plus `SLUG_PATTERN` / `SLUG_REGEX` / `SLUG_MIN_LEN` / `SLUG_MAX_LEN` constants. No new runtime deps — uses stdlib `re` + `unicodedata` only.
- `slugify` uses NFKD normalization + ASCII encode-ignore for best-effort transliteration (Émilie → emilie, Café → cafe, Résumé → resume). Emoji and non-Latin scripts are dropped entirely — English-only by design per CLAUDE.md. Leading digits/underscores are stripped so `"123 Notes Daily"` → `"notes_daily"` and `"_foo"` → `"foo"`. Trailing underscores are stripped before and after the 40-char truncation so truncating mid-run never leaves a dangling separator.
- `resolve_unique_slug` detects collisions by directory (or file) presence at `skills_dir/<candidate>`; a stray regular file sitting where a skill dir would go also blocks the slug (overwriting random files is worse than choosing another slug). Raises `SlugError` if the numeric suffix would push the candidate past `SLUG_MAX_LEN` — unreachable for sensible bases but worth being explicit.
- `validate_user_slug` returns `(True, trimmed_slug)` or `(False, human_reason)`. Reason strings name the specific defect (too short, too long, uppercase, starts-with-digit, disallowed chars with the offenders listed, already-exists). UI can surface these verbatim.

- **Learnings for future iterations:**
  - `unicodedata.normalize("NFKD", s).encode("ascii", "ignore").decode("ascii")` is the stdlib pattern for transliteration — no Unidecode dep needed. It splits combining diacritics off their base letters, then the ASCII-ignore encode drops the marks (and every non-ASCII codepoint, including emoji, CJK, etc.). Downstream code that needs transliteration should reuse this idiom — it's about 5 lines, not worth a helper.
  - `str.rstrip("_")` after truncation is mandatory — slicing `s[:40]` can leave a trailing `_` if position 40 happened to be a separator, and the resulting slug would fail `SLUG_REGEX`. The slugify tests hit this boundary explicitly (`"a"*39 + " " + "b"*10`).
  - The pattern `^[a-z][a-z0-9_]{2,39}$` bounds TOTAL length to 3–40 (first char + 2–39 more). A common off-by-one would be to think "2 to 39 more" means total 2–39. The tests at the min/max boundaries (`"abc"` and `"a" + "b"*39`) exist to pin this down.
  - `SlugError` subclasses `ValueError` so callers that catch `ValueError` broadly still see it; keep it as a specific subclass so new call sites can tighten up.
  - Ruff's `RUF059` flags unused unpacked tuple members (`ok, reason = ...` when only `ok` is asserted). The fix is to either prefix with `_` or actually use the variable; the test suite now asserts on the reason text wherever it's unpacked, which doubles as a sanity check on the user-facing message quality.
  - Ruff's `RUF002` flags `–` (EN DASH) and `—` (EM DASH) in docstrings as ambiguous with hyphen-minus. Prose in code docstrings should use ASCII `-` or switch to EM DASH only where clearly a sentence break; raw EN DASH in ranges like "3–40" must be rewritten as "3-40". (The synthesizer package already has EM DASH uses in existing docstrings that ruff has been accepting — only EN DASH and the multiplication `×` glyph trip the checker. The E501 allowance on `draft_prompt.py` does NOT help here.)

## 2026-04-22 - S-005 Trajectory preprocessor: collapsing, downsampling, and keyframe selection
- Added `services/synthesizer/src/synthesizer/preprocess.py` with `preprocess_trajectory(reader) -> PreprocessedTrajectory`. Three-stage reduction: noise filter (drops mouse_move/mouse_down/mouse_up), collapse (scroll runs within 3s same-app windows → `scroll_run`; ≥5s gaps between retained events → `idle`), keyframe selection capped at 20 using priority order (app_switch → pre-click-cluster → equal-time-interval fillers).
- `DigestEntry` and `PreprocessedTrajectory` are pydantic v2 frozen models mirroring `ParsedSkill`/`Event` conventions. Payload dict is kind-specific: `scroll_run` carries `{total_delta, duration_ms}`, `idle` carries `{duration_ms}`. Frozen guarantees downstream prompt builders can cache/hash entries safely.
- Deterministic summary text: clicks include `target.label` + app name (e.g., `Clicked button labeled "Send" in Chrome`), text_inputs quote the typed text truncated at 60 chars, app_switches/scrolls/key events all produce 1-sentence summaries with no LLM call. `APP_NAMES` table maps common macOS bundle_ids to human names.
- Token estimator: `text_chars / 4 + screenshots * 1500`. Rough but tight enough for prompt-fit decisions; the test pins it within 20% of a hand-computed expectation.
- Added `tests/synthesizer/test_preprocess.py` with 15 tests covering every PRD acceptance item: 50-scroll collapse, scroll breaks on app-change / long-gap, idle injection at ≥5s gap, idle not injected at <5s, 30 clicks + 10 text_inputs preserved, mouse noise dropped, 100-keyframe cap at 20, app_switch priority-1 selection, click-cluster priority-2 selection, all-screenshots-kept-when-under-cap, fixture happy path, token estimate bounds, empty trajectory, frozen-model mutation raises.
- Files changed: services/synthesizer/src/synthesizer/preprocess.py (new), tests/synthesizer/test_preprocess.py (new), scripts/ralph/prd.json (passes: true for S-005), scripts/ralph/progress.txt (this entry + patterns).
- Quality gates: `uv run ruff check services/synthesizer tests/synthesizer` ✓, `uv run mypy --strict services/synthesizer` ✓ (7 source files), `uv run pytest tests/synthesizer/` ✓ (129 passed, +15 from S-004).
- **Learnings for future iterations:**
  - Tests that build a synthetic trajectory directly in `tmp_path` must supply every required metadata field (id, started_at, stopped_at, label, display_info, app_focus_history) or the `TrajectoryReader` constructor fails before preprocessing is reached — shared `_make_trajectory` helper handles this and is a candidate for a conftest fixture if S-008/S-010 need the same shape.

## 2026-04-22 - S-006 Anthropic client wrapper with cost logging, retry, and fake mode

- `LLMClient.complete(*, messages, system, tools=None, max_tokens=4096, model='claude-sonnet-4-5', context_label='')` returns a normalized `LLMResponse(text, stop_reason, input_tokens, output_tokens, raw)` and writes one line to `costs.jsonl` under the active profile dir per call (real or fake).
- Real-mode retry policy: catch `anthropic.APIStatusError`, retry on 429/500/502/503/504 with backoff (1s, 2s, 4s, 8s, 16s) up to 5 retries; SDK retries disabled (`max_retries=0`) so the wrapper owns the schedule. `anthropic.AuthenticationError` (401) raises `ConfigurationError` immediately, no retry. Connection / timeout errors share the backoff schedule. Sleep is injectable for test speed.
- Fake mode (default in tests via the autouse env var): `compute_request_hash(...)` over canonical JSON of the inputs determines the file name under the fake-responses dir; canned JSON files have keys `{text, stop_reason, input_tokens, output_tokens}`. - Costs path resolution: `TRACE_DATA_DIR` env override (tests) > `TRACE_PROFILE` selecting `Trace` (prod) vs `Trace-dev` (dev), default prod. New `trace_data_dir()` and `costs_log_path()` helpers exposed for reuse.

- **Learnings for future iterations:**
  - The `_force_fake_mode` autouse fixture in `tests/synthesizer/conftest.py` sets `TRACE_LLM_FAKE_MODE=1` for every test. To exercise the real HTTP path (still mocked via respx), use the `real_mode` fixture in `test_llm_client.py` as a pattern — it does `monkeypatch.delenv("TRACE_LLM_FAKE_MODE")`. Future tests for `draft.py`/`revise.py` retry behavior can copy this fixture or import it via conftest.
  - Fake-mode is checked at *call* time, not constructor time — this matters because the autouse fixture sets the env var BEFORE the test body runs. If you bind the mode at construction, then changing the env in the test won't take effect on already-constructed clients.
  - `respx.MockRouter.side_effect` accepts a list of `httpx.Response` objects and consumes them one per call — perfect for retry sequences. Once exhausted it raises; size your list to the expected call count + 0 to fail loudly on extra attempts.
  - The Anthropic SDK's `Anthropic(api_key=..., max_retries=0)` is the right way to disable SDK retries; passing `0` (not `None`) is required since `None` means "use default".
  - When `respx.MockRouter.assert_all_called=False` is used (set in conftest), routes can be created and never hit without test failure — useful for the autouse network guard.
  - `httpx.Response(503, json=...)` with the Anthropic error envelope `{"type":"error","error":{"type":"overloaded","message":"..."}}` is what the SDK expects; minimal-shape JSON is enough for `APIStatusError.status_code` to populate.
