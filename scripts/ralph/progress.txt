## Codebase Patterns
- uv workspace: dev extras on workspace members aren't picked up by `uv sync --extra dev` at the root; use `uv sync --all-extras --all-packages` (or run per-package) to install a workspace member's dev deps.
- Synthesizer package uses **src layout** (`services/synthesizer/src/synthesizer/`) with hatchling; declare `[tool.hatch.build.targets.wheel] packages = ["src/synthesizer"]` so the wheel finds it.
- `pytest` exits 5 on "no tests collected" — that's NOT zero. If a story's AC requires `pytest ... exits 0`, add at least one smoke test to the new suite.
- All synthesizer tests run with `TRACE_LLM_FAKE_MODE=1` + `ANTHROPIC_API_KEY=test-dummy-key` forced via an autouse fixture in `tests/synthesizer/conftest.py`, plus the `anthropic_mock` respx router is autouse-installed on every test so an unmocked call to `api.anthropic.com` raises instead of hitting the network.
- Per-service `pyproject.toml` declares its own `[tool.mypy] strict = true` so `uv run mypy --strict services/synthesizer` uses service-scoped config (not the root's). Root `[tool.mypy]` still covers `services` + `gateway` for repo-wide checks.
