## Codebase Patterns
- Runner uses a **src layout** (`services/runner/src/runner/`). Tests import `from runner.xxx`, resolved via `pythonpath = ["services/runner/src"]` in the root `pyproject.toml` pytest config. Do NOT add a nested `services/runner/runner/` package; that was the old flat layout and has been removed.
- The root pyproject owns the shared `ruff`, `mypy`, and `pytest` config. The runner's own `pyproject.toml` only adds a mypy `files = ["src"]` override so `uv run mypy --strict services/runner` targets the runner package directly.
- Safety gate: every live-mode code path must call `runner.safety.require_live_mode()` in `__init__`. The gate is strict string equality against `TRACE_ALLOW_LIVE="1"` ŌĆö `"true"`, `"yes"`, `"0"`, etc. all disable live mode. `tests/runner/conftest.py` autouses a fixture that force-unsets the flag before every test; opt in with the `live_mode_allowed` fixture.
- Tests mock Anthropic via `respx` (see `anthropic_mock` fixture) ŌĆö never hit the real API.
- Gateway (`gateway/main.py`) imports the runner as `from runner.api import router` (installed-package style via the pytest pythonpath). Synthesizer/recorder are still on the flat layout and import via `from services.{name}.{name}.api` ŌĆö those packages will switch to src layout when their respective branches reach X-001-equivalent stories.
- JSON schemas in `contracts/` are the on-disk contract; the runner pairs each with a pydantic model in `runner.schema` (e.g. `RunMetadata`) for typed in-memory use. Both must agree ŌĆö `extra="forbid"` on the model mirrors `additionalProperties: false` on the schema. Round-tripping uses `model_dump(mode="json", exclude_unset=True)` so unset optional fields do not leak back as nulls.
- Pydantic v2 serializes UTC datetimes with a trailing `Z` (not `+00:00`) via `mode="json"`. Tests that assert round-trip identity must store the canonical `Z` form in fixtures.
- `types-jsonschema` is in the shared dev extras so mypy --strict accepts `jsonschema` imports without module-level overrides.
- Shared contracts that need validation beyond JSON Schema (e.g. list-length, lowercase) live at the repo root under `contracts/` and are policed by `scripts/check_contracts.sh`. Runtime code loads them through a module in `runner.*` with `@lru_cache(maxsize=1)`; the schema path is computed from `Path(__file__).resolve().parents[4]` (for `services/runner/src/runner/*.py`).
- Non-ASCII literals in tests (fullwidth letters, NBSP) should be written as `\uXXXX` escape sequences, not pasted as glyphs ŌĆö ruff's RUF001 flags ambiguous characters and breaks lint otherwise.
- The synthesizer package is now importable at the top level as `synthesizer.*` (not `services.synthesizer.synthesizer.*`). `services/synthesizer` is on both pytest `pythonpath` and mypy `mypy_path`. The recorder is still flat-layout and reached as `services.recorder.recorder.*` until its own X-001-equivalent story lands. Gateway uses `from synthesizer.api import ...` to match.
- Cross-module typechecking: leaving `services/synthesizer/synthesizer` out of mypy `files` while keeping it on `mypy_path` avoids the "source file found twice" ambiguity. Synthesizer code still gets strict-checked because the runner imports it and mypy follows those imports.
- SKILL.md format (locked by X-004 parser): a `# Title` line, a `## Steps` section whose body is `N. text` lines (1-based, contiguous). Destructive steps begin with `ŌÜĀ’ĖÅ ` (fully qualified) or `ŌÜĀ ` (bare). The parser strips the marker from `.text` and sets `destructive=True`. Prose between title and `## Steps` is ignored but preserved on disk.
- Skill meta/markdown semantic check lives in `synthesizer.schema.validate_meta_against_markdown` ŌĆö it enforces that `meta.destructive_steps` is exactly the set of `ŌÜĀ’ĖÅ`-marked step numbers (symmetric: either-direction drift is an error).
- Parameter substitution: `meta.parameters[*]` is the declaration. A parameter is required iff it has NO `default` key at all OR `default` is `null`. Unknown parameters from the caller raise `UnknownParameterError`. Escaped braces `\{` / `\}` survive substitution as literal `{` / `}`. Non-string defaults (e.g. integer `30`) are stringified via `str()` before substitution.
- Atomic metadata writes: `runner.run_writer._atomic_write_bytes` uses `os.open` + `os.write` + `os.fsync` + `os.replace` with a `.tmp` sibling, then removes the tmp on any exception. Always patch `runner.run_writer.os.fsync` (not `os.fsync`) in tests so the module-local reference is intercepted. `update_status` re-runs pydantic validation via `model_dump() | updates ŌåÆ model_validate` ŌĆö `model_copy(update=...)` does NOT re-validate and will happily accept a bogus status enum.
- Retina geometry: `runner.coords` exposes `points_to_pixels`/`pixels_to_points` (factor-of-`scale_factor`) and a combined `resized_pixels_to_points(x, y, mapping)` where `mapping.scale_from_resized_to_points = (original_w / resized_w) / scale_factor` ŌĆö use `ImageMapping` to convert Claude-space coordinates back to display points in one multiply. `capture_and_normalize` is pure normalization; the `source_image=None` default raises `NotImplementedError` because live capture is X-009's `LiveScreenSource`. Use `DryRunDisplayInfo` (2880x1800 / 1440x900 / 2.0) as the canonical dry-run display. `round(x)` returns `int` directly ŌĆö never wrap in `int(round(...))` (triggers RUF046).

## 2026-04-22 - X-005 - Run artifact writer with atomic metadata updates and screenshot storage
- Added `services/runner/src/runner/run_writer.py` exposing `RunWriter` (thread-safe via a single `RLock`) with `write_metadata`, `update_status`, `append_event`, `append_transcript`, `write_screenshot`, `close`, and a context-manager protocol. Also exported `get_run_dir_perms` as a tiny test helper.
- Atomic metadata writes use `os.open` + `os.write` + `os.fsync` + `os.replace` with a `.tmp` sibling and unlink-on-exception to guarantee no stray tmp files after a mid-write crash. The metadata fast-path short-circuits when the serialized bytes match the last successful write ŌĆö same-contents `write_metadata` is a true no-op, preserving mtime.
- `update_status` reconstructs the model via `model_dump() | updates` ŌåÆ `RunMetadata.model_validate(...)` so a bad status enum fails loudly. `model_copy(update=...)` does NOT re-validate in pydantic v2 and would have silently accepted nonsense.
- Screenshot writer validates the PNG magic bytes `89 50 4E 47 0D 0A 1A 0A` before writing, rejects negative seq, and zero-pads to 4 digits via `_screenshot_filename`.
- Run directory is created with mode `0o700` (matching the safety invariant) and an `os.chmod` after `mkdir` ŌĆö `Path.mkdir(mode=...)` honors umask, so an explicit chmod is required.
- Added `tests/runner/test_run_writer.py` ŌĆö 16 cases covering: directory perms 0o700, full lifecycle (10 events + 5 screenshots + 3 transcript turns), metadata idempotency, runid/slug/mode mismatch ŌåÆ ValueError, invalid status enum ŌåÆ ValidationError, concurrent append from 3 threads producing 30 monotonic-seq events, simulated `fsync` failure (patched at `runner.run_writer.os.fsync`) leaves no `.tmp` file AND preserves prior metadata, PNG magic rejection, negative seq rejection, zero-padding, `close()` idempotency, context-manager, individual JSONL line parseability, transcript content-block preservation.
- Quality gates: `uv run ruff check services/runner tests/runner` clean; `uv run mypy --strict services/runner` clean (8 source files); `uv run pytest tests/runner/` ŌåÆ 174 passed (16 new); `scripts/check_contracts.sh` ok.
- **Learnings for future iterations:**
  - Pydantic v2: `model_copy(update=...)` does NOT re-run field validators. To enforce schema integrity when mutating a RunMetadata, use `model_validate(model_dump() | updates)` instead.
  - `os.fsync` is imported into `runner.run_writer` at module level for atomic writes. Tests that want to simulate a fsync failure MUST patch `runner.run_writer.os.fsync` (the module-local reference), not `os.fsync` globally ŌĆö otherwise the patch won't intercept.
  - `Path.mkdir(mode=0o700)` is subject to the process umask; always follow with an explicit `os.chmod(path, 0o700)` to guarantee the perms actually land on disk.
  - Minimal valid PNG bytes for tests: 8-byte signature + IHDR + IDAT + IEND chunks ŌĆö about 67 bytes total. Stored inline in the test module to avoid shipping a fixture file just for the happy path.
