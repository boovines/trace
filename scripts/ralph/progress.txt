## Codebase Patterns
- Runner uses a **src layout** (`services/runner/src/runner/`). Tests import `from runner.xxx`, resolved via `pythonpath = ["services/runner/src"]` in the root `pyproject.toml` pytest config. Do NOT add a nested `services/runner/runner/` package; that was the old flat layout and has been removed.
- The root pyproject owns the shared `ruff`, `mypy`, and `pytest` config. The runner's own `pyproject.toml` only adds a mypy `files = ["src"]` override so `uv run mypy --strict services/runner` targets the runner package directly.
- Safety gate: every live-mode code path must call `runner.safety.require_live_mode()` in `__init__`. The gate is strict string equality against `TRACE_ALLOW_LIVE="1"` â€” `"true"`, `"yes"`, `"0"`, etc. all disable live mode. `tests/runner/conftest.py` autouses a fixture that force-unsets the flag before every test; opt in with the `live_mode_allowed` fixture.
- Tests mock Anthropic via `respx` (see `anthropic_mock` fixture) â€” never hit the real API.
- Gateway (`gateway/main.py`) imports the runner as `from runner.api import router` (installed-package style via the pytest pythonpath). Synthesizer/recorder are still on the flat layout and import via `from services.{name}.{name}.api` â€” those packages will switch to src layout when their respective branches reach X-001-equivalent stories.
