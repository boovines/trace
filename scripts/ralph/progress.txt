## Codebase Patterns
- `services/recorder` uses a **src layout** (`services/recorder/src/recorder/â€¦`); future files (schema.py, writer.py, session.py, etc.) go under `src/recorder/`.
- The recorder package is installed as a uv workspace member (`trace-recorder`) and should be imported as `from recorder.xyz import â€¦`, NOT `services.recorder.recorder.xyz`.
- Root `pyproject.toml` owns shared tooling config (ruff/mypy/pytest); per-service `pyproject.toml` only declares that service's runtime deps + hatchling build.
- Ruff's isort needs `known-first-party = ["recorder", "synthesizer", "runner", "gateway"]` or it misgroups the workspace imports in gateway/main.py.
- PyObjC wheels only exist for macOS â€” deps are marked `; sys_platform == 'darwin'` so `uv sync` still works if Ralph ever runs on non-darwin.
- `mypy --strict` has PyObjC module overrides in root pyproject (`Cocoa.*`, `Quartz.*`, `ApplicationServices.*`, `AppKit.*`, `Foundation.*`, `objc.*`); add new PyObjC module imports here, not in each file.
- Quality-gate commands for the recorder module are:
  - `uv run ruff check services/recorder`
  - `uv run mypy --strict services/recorder`
  - `uv run pytest tests/recorder/`
- Pre-existing gateway/integration test failures (`ModuleNotFoundError: gateway`) are NOT caused by R-001 and are outside this branch's scope â€” a future `feat/integration` story will install gateway as a proper package.
- **Schema validation**: `recorder.schema` exposes `validate_event(event)` / `validate_metadata(meta)` backed by `jsonschema`'s Draft 2020-12 validator (with `FORMAT_CHECKER` enabled for `uuid`/`date-time`). Future writer/session code should call these before writing JSON to disk so bad events never land in `events.jsonl`.
- **Schema file lookup**: `recorder.schema.find_schema_path()` walks up from `__file__` to find `contracts/trajectory.schema.json`. Works for the uv-workspace editable install. If the contracts ever move or get bundled into the wheel, update this function â€” do NOT hardcode a path.
- **Local `$ref` only**: the `load_event_schema()` / `load_metadata_schema()` wrappers build standalone schemas with a `#/$defs/...` fragment ref (no `$id`), which keeps `jsonschema` from trying to HTTP-fetch the schema's `$id` URL at validate time. If you add new top-level sub-schemas for Synthesizer/Runner to reuse, follow the same pattern.
- **TrajectoryWriter ownership**: `recorder.writer.TrajectoryWriter` is the only module allowed to mutate the on-disk trajectory directory. All writes go through its lock; session/orchestrator code must not write `events.jsonl` / `metadata.json` / `screenshots/*` directly. It validates via `recorder.schema.validate_{event,metadata}` BEFORE any disk write, so bad data never leaks to disk. The constructor creates `<root>/<uuid>/` + `screenshots/` with perms `0700` and the writer is usable as a context manager (`__exit__` calls `close()`).
- **Atomic JSON writes**: `_atomic_write_json(path, data)` writes to `<path>.tmp`, fsyncs, and renames. Use it for any metadata-style file. `events.jsonl` is append-flush (no fsync per line â€” documented v1 tradeoff).
- **Datetime import style**: use `from datetime import UTC, datetime` (Python 3.11+). Ruff's `UP017` flags `timezone.utc` â€” don't revive it. ISO strings are produced with `datetime.now(UTC).isoformat(timespec="milliseconds")` which yields `+00:00` suffixes; the schema's `format: date-time` accepts both `+00:00` and `Z`.
- **Ruff SIM105**: prefer `with contextlib.suppress(OSError): os.fsync(...)` over `try/except/pass` for best-effort fsync on filesystems that don't support it (tmpfs in pytest tmp_path).
- **PyObjC mocking in tests**: for any module that does `from ApplicationServices import X` / `from Quartz import Y` inside a function, tests stub the framework by installing a `types.ModuleType("Quartz")` / `types.ModuleType("ApplicationServices")` into `sys.modules` via `monkeypatch.setitem`. The import inside the tested function then resolves to the stub. For ImportError coverage, install a stub that is missing the expected attributes â€” `from X import missing_attr` raises ImportError, which the module under test should catch.
- **Permission-check pattern**: `recorder.permissions` keeps each macOS framework call in its own thin `_check_*` helper, and `check_permissions()` / `get_missing_permissions_error()` simply compose them. Tests patch the helpers for combination coverage and patch the PyObjC symbols inside `sys.modules` for helper-level coverage. Use the same pattern for any future PyObjC-backed module (event_tap, ax_resolver, focus_tracker, screenshot) to keep the suite hermetic on Ralph's sandbox.
- **`@pytest.mark.macos` is opt-in only**: `tests/recorder/conftest.py` auto-skips every test tagged `macos` unless `TRACE_RUN_MACOS_TESTS=1` is set. Use this marker for ANY test that needs real PyObjC framework behaviour (CGEventTap, AXUIElement, CGWindowListCreateImage, etc.). Hermetic unit coverage should still be provided alongside via `sys.modules`-stubbed Quartz/ApplicationServices so the default run stays green.
- **Splitting PyObjC callback from normaliser**: for modules driven by a C-callback (CGEventTap), put the pure eventâ†’dict conversion in a module-level function that takes the framework module as a parameter (`_normalise_event(code, event, quartz)`), and expose a thin `handle_cg_event(code, event)` method on the class that the tap callback delegates to. Tests exercise the normaliser + re-enable logic by calling `handle_cg_event` with fake events â€” no run-loop or thread needed. `event_tap.py` is the reference implementation.
- **Re-enabling a disabled tap**: `CGEventTapEnable(tap, True)` re-arms a tap the OS disabled via `kCGEventTapDisabledByTimeout` / `kCGEventTapDisabledByUserInput`. Both constants appear as unsigned `uint32` (0xFFFFFFFE / 0xFFFFFFFF) in headers but PyObjC sometimes surfaces them as signed ints (-2 / -1); match both forms.

- **Learnings for future iterations:**
  - `datetime.now(timezone.utc)` trips ruff `UP017` on Python 3.11+ â€” use `from datetime import UTC` and `datetime.now(UTC)`.
  - `tmp.replace(final)` on POSIX is atomic and works cross-device on the same filesystem; it's the right primitive for "write then swap into place".

  - `close()` reading the metadata from disk as a fallback means that even if a test calls `close()` without ever calling `write_metadata()`, we don't crash â€” we just skip the `stopped_at` finalisation. The orchestrator should always call `write_metadata()` first, but the writer defends against the bad ordering.
